  <name>Task 1: Meetings sync endpoint</name>
  <files>backend/src/flywheel/api/meetings.py</files>
  <action>
Create new file `backend/src/flywheel/api/meetings.py` with a single endpoint.

**Router:**
```python
router = APIRouter(prefix="/meetings", tags=["meetings"])
```

**POST /sync endpoint (`sync_meetings`):**

1. **Find Granola integration:** Query Integration where tenant_id=user.tenant_id, user_id=user.sub, provider="granola", status="connected". Return 400 with "Granola not connected. Add your API key in Settings." if not found.

2. **Decrypt API key:** Use `decrypt_api_key(integration.credentials_encrypted)` from `flywheel.auth.encryption`.

3. **Fetch meetings:** Call `list_meetings(api_key, since=integration.last_synced_at)` from `flywheel.services.granola_adapter`.

4. **Dedup:** If raw_meetings is non-empty, SELECT existing external_ids from meetings table where tenant_id + provider="granola" + external_id IN (list of fetched ids). Build a set of already-known IDs.

5. **Insert new meetings:** For each raw meeting not in existing_ids:
   - Apply processing rules via `_apply_processing_rules(raw, rules)` helper (returns "pending" or "skipped")
   - Create Meeting row with: tenant_id=user.tenant_id, user_id=user.sub, provider="granola", external_id, title, meeting_date, duration_mins, attendees, ai_summary, processing_status
   - Count synced (pending) and skipped separately

6. **Update cursor:** Set `integration.last_synced_at = datetime.now(timezone.utc)`

7. **Commit and return:**
```python
return {
    "synced": synced,
    "skipped": skipped,
    "already_seen": len(existing_ids),
    "total_from_provider": len(raw_meetings),
}
```

**Helper function `_apply_processing_rules(raw: RawMeeting, rules: dict) -> str`:**
- If `rules.get("skip_internal")` is True and all attendees have `is_external=False` -> return "skipped"
- If `rules.get("min_duration_mins")` is set and raw.duration_mins is not None and raw.duration_mins < threshold -> return "skipped"
- If `rules.get("skip_domains")` is a list and all attendee emails match any skip domain -> return "skipped"
- Default: return "pending"

Use `user.sub` for user_id (NOT `user.id`) per Phase 59 decision.
