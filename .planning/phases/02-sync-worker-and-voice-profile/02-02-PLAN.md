**`VOICE_SYSTEM_PROMPT` constant:**
```
You are analyzing email samples to extract a user's writing voice profile.
Return a JSON object with exactly these fields:
- tone: string (e.g., "professional and concise", "warm and conversational")
- avg_length: integer (estimated average email length in words)
- sign_off: string (most common sign-off phrase, e.g., "Best," or "Thanks,")
- phrases: array of strings (3-5 characteristic phrases or expressions the person uses)
- samples_analyzed: integer (number of emails you analyzed)

Return only the JSON object, no other text.
```

**`_HAIKU_MODEL = "claude-haiku-4-5-20251001"`**

**`voice_profile_init(db, integration)` — main entry point:**
- Check if `EmailVoiceProfile` already exists for `(tenant_id, user_id)` — if yes, return False (idempotent guard)
- Get credentials via `get_valid_credentials(integration)`
- Call `list_sent_messages(creds, max_results=200)` from gmail_read to get sent message stubs
- Iterate stubs, for each: if `len(substantive_bodies) >= 100` then break. Otherwise call `get_message_body(creds, stub["id"])` from gmail_read, apply `_is_substantive(body)`, append to `substantive_bodies` if passes
- If fewer than 3 substantive bodies found: log warning, return False (not enough signal)
- Call `_extract_voice_profile(substantive_bodies[:20])` — send only the 20 most recent to control token cost
- Upsert `EmailVoiceProfile` using `pg_insert().on_conflict_do_update(constraint="uq_voice_profile_tenant_user")` with fields: tenant_id, user_id, tone, avg_length, sign_off, phrases, samples_analyzed
- Commit and return True
- Log at INFO level: "Voice profile created for integration %s (%d samples analyzed)" with integration.id and samples count

**`_is_substantive(body)` — filter function:**
- Return False if body is None, empty, or `len(body.strip()) < 50`
- Check against auto-reply patterns (case-insensitive): "out of office", "auto-reply", "automatic reply", "i am currently out", "accepted:", "declined:", "tentative:", "has accepted your invitation", "this is an automated", "do not reply to this", "i'm ooo", "i will be out"
- Return False if any pattern matches
- Count sentences: `re.split(r'[.!?]\s+', body.strip())`, filter to sentences with `len(s.strip()) > 10`
- Return `len(sentences) >= 3`

**`_extract_voice_profile(bodies)` — LLM call:**
- Uses `anthropic.AsyncAnthropic(api_key=settings.flywheel_subsidy_api_key)` — same pattern as onboarding_streams.py
- `client.messages.create(model=_HAIKU_MODEL, max_tokens=1000, system=VOICE_SYSTEM_PROMPT, messages=[{"role": "user", "content": "\n\n---\n\n".join(bodies)}])`
- Parse response: `response.content[0].text.strip()`
- Try `json.loads(text)`. On `JSONDecodeError`: regex fallback `re.search(r"\{.*\}", text, re.DOTALL)` then `json.loads(match.group())`. If no match, re-raise.
- Return the parsed dict

Modify the `_sync_one_integration()` function (created in Plan 02-01) to call `voice_profile_init` after a successful `sync_gmail` call.

The wiring logic inside `_sync_one_integration`:

```python
# After sync_gmail succeeds:
count = await sync_gmail(session, integration)

# Check if voice profile needs initialization (runs once per user)
existing_profile = await session.execute(
    select(EmailVoiceProfile).where(
        and_(
            EmailVoiceProfile.tenant_id == integration.tenant_id,
            EmailVoiceProfile.user_id == integration.user_id,
        )
    )
)
if existing_profile.scalar_one_or_none() is None:
    try:
        created = await voice_profile_init(session, integration)
        if created:
            logger.info("Voice profile initialized for integration %s", integration.id)
    except Exception:
        logger.exception("Voice profile init failed for integration %s", integration.id)
        # Non-fatal — sync continues even if voice init fails
```

Key design decisions:
- The `EmailVoiceProfile` existence check happens in the CALLER before calling voice_profile_init, to avoid the unnecessary function call overhead on every cycle. The function also has its own idempotency guard (belt and suspenders).
- Voice profile failure is caught and logged but does NOT prevent email sync from being considered successful — it's a best-effort enhancement.
- This runs inside the tenant_session context, so RLS is enforced for the voice profile upsert.
- The check uses the already-imported `select`, `and_`, and `EmailVoiceProfile` — no new imports needed beyond what Task 1 added.
