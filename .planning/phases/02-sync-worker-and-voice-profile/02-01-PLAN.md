  <name>Task 1: Create gmail_sync.py with email_sync_loop and sync_gmail</name>
  <files>backend/src/flywheel/services/gmail_sync.py</files>
  <action>
Create `backend/src/flywheel/services/gmail_sync.py` modeled directly on `calendar_sync.py`. The file implements:

**Constants:**
- `SYNC_INTERVAL = 300` (5 minutes)
- `PER_INTEGRATION_TIMEOUT = 60.0` (seconds, for asyncio.wait_for)

**`email_sync_loop()` — the main background worker:**
- `while True` loop with `asyncio.sleep(SYNC_INTERVAL)` at the bottom
- Uses `get_session_factory()` then `factory() as session` (superuser context) to query all `Integration` rows where `provider="gmail-read"` and `status="connected"`
- For EACH integration, wraps `_sync_one_integration(factory, intg)` in `asyncio.wait_for(..., timeout=PER_INTEGRATION_TIMEOUT)` and collects into `asyncio.gather(*tasks, return_exceptions=True)` — this satisfies GMAIL-08 concurrent polling
- After gather, iterate results: log TimeoutError as warning, log other exceptions via `logger.exception`, log successful counts at INFO level
- Outer try/except around entire iteration body logs and swallows — never crash the loop

**`_sync_one_integration(factory, integration)` — per-integration wrapper:**
- Opens a `tenant_session(factory, str(integration.tenant_id), str(integration.user_id))` context for RLS
- Re-loads the integration row inside the tenant session (the one from the outer session is detached)
- Calls `sync_gmail(session, integration)` and returns the count
- Catches `TokenRevokedException` from gmail_read — sets `integration.status = "disconnected"`, `integration.credentials_encrypted = None`, commits, logs warning

**`sync_gmail(db, integration, _retry_count=0)` — incremental or full sync:**
- Gets credentials via `get_valid_credentials(integration)` from gmail_read
- Reads `history_id = (integration.settings or {}).get("history_id")`
- If `history_id is None`: call `_full_sync(db, integration, creds)` and return
- Try `get_history(creds, history_id)` from gmail_read
- Catch `HttpError`: if `exc.resp.status == 404 and _retry_count < 1`, set `settings["history_id"] = None` on integration, recursive call with `_retry_count=1`. Otherwise re-raise.
- Process response: `response.get("history", [])` (NOT `response["history"]` — Gmail omits the key when empty)
- Extract message IDs from `record.get("messagesAdded", [])` entries
- For each message_id: call `get_message_headers(creds, message_id)` and then `upsert_email(db, integration.tenant_id, integration.user_id, msg)`
- Update `historyId`: `response.get("historyId")` — store in `integration.settings["history_id"]`
- Set `integration.last_synced_at = datetime.now(timezone.utc)`
- `await db.commit()` once after all upserts
- Return count of messages processed

**`_full_sync(db, integration, creds)` — initial or recovery sync:**
- CRITICAL: Call `get_profile(creds)` FIRST to capture `historyId` BEFORE pagination begins — this prevents missing messages that arrive during pagination
- Paginate via `list_message_headers(creds, page_token=..., label_ids=["INBOX"], max_results=100)`
- For each message stub: `get_message_headers(creds, stub["id"])` then `upsert_email(db, ...)`
- After pagination complete: store `checkpoint_history_id` from get_profile into `integration.settings["history_id"]`
- Set `integration.last_synced_at`, commit, return count

**`upsert_email(db, tenant_id, user_id, msg)` — PostgreSQL upsert:**
- Extract headers from `msg.get("payload", {}).get("headers", [])` into a dict
- Parse `Date` header using `dateutil.parser.parse()` (already installed as transitive dep)
- Extract sender_email using simple split on `<`/`>` from `From` header (use `email.utils.parseaddr` from stdlib)
- Build values dict: `tenant_id, user_id, gmail_message_id=msg["id"], gmail_thread_id=msg["threadId"], sender_email, sender_name, subject=headers.get("Subject"), snippet=msg.get("snippet"), received_at, labels=msg.get("labelIds", []), is_read="UNREAD" not in msg.get("labelIds", [])`
- Use `pg_insert(Email).values(**values).on_conflict_do_update(constraint="uq_email_tenant_message", set_={...})` — update subject, snippet, labels, is_read, synced_at on conflict
- Do NOT commit inside this function — caller commits after batch

**`_parse_email_date(date_str)` helper:**
- Uses `email.utils.parsedate_to_datetime` from stdlib (handles RFC 2822 email dates natively)
- Returns None on parse failure (not crash)

**Logging rules (CRITICAL — PII compliance):**
- NEVER log msg dict, subject, snippet, sender_email, or body content
- Only log: message_id, thread_id, integration.id, counts
- Use `logger = logging.getLogger(__name__)`

**Imports needed:**
- `from flywheel.services.gmail_read import get_valid_credentials, get_history, list_message_headers, get_message_headers, get_profile, TokenRevokedException`
- `from flywheel.db.models import Email, Integration`
- `from flywheel.db.session import get_session_factory, tenant_session`
- `from sqlalchemy.dialects.postgresql import insert as pg_insert`
- `from sqlalchemy import and_, select, func`
- `from googleapiclient.errors import HttpError`
- `import email.utils` (for parseaddr and parsedate_to_datetime)
