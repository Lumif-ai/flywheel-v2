  ```typescript
  { title: trimmed, task_type: 'other', commitment_direction: 'yours', priority: 'medium', trust_level: 'review', due_date: new Date().toISOString().split('T')[0] }
  ```
  Note: due_date as YYYY-MM-DD string (date only, not full ISO with time).
- On success callback: clear input, invalidate `['briefing-v2']` via `queryClient.invalidateQueries({ queryKey: ['briefing-v2'] })`.
- Use Plus icon (lucide-react, size 16) as a left adornment inside the input wrapper via absolute positioning.

**TaskRow (inline in same file):**
- Layout: flex row, items-center, gap-3, py-2.5, border-b last:border-0.
- Checkbox: A custom div styled as a checkbox (20x20, rounded-md, border 1.5px). When unchecked: border colors.subtleBorder, bg transparent. On click: set local `checked` state to true immediately (optimistic), call `useUpdateTaskStatus().mutate({ id: task.id, status: 'done' })` with:
  - `onError`: set checked back to false
  - `onSettled`: `queryClient.invalidateQueries({ queryKey: ['briefing-v2'] })`
  When checked: bg coral (#E94D35), border coral, show Check icon (lucide-react, size 14, white) inside. Add a brief transition (150ms) for the fill.
  
  **Important (from research Pitfall 2):** Do NOT try to write optimistic updates into the briefing-v2 cache. Use local component state for instant visual feedback, then let the settled invalidation refresh real data.

  **Task status handling (from research):** For tasks with status `detected` or `in_review`, transition to `confirmed` first. For `confirmed` or `in_progress`, transition directly to `done`. Check `VALID_TRANSITIONS` from tasks/types/tasks.ts. Simplification: if task.status is `confirmed` or `in_progress`, set to `done`. If `detected`, set to `confirmed`. If `in_review`, set to `confirmed`. This avoids backend rejection.

- Due date (if set): 12px, colors.secondaryText. Format with date-fns: if overdue (isBefore today), show in red (colors.error) with text like "Apr 5" or "Overdue". If today, show "Today". Otherwise show formatted date.
- Source badge: Use the Badge component (variant="secondary") showing the source value. Map source strings to display labels: "manual" -> "Manual", "meeting" -> "Meeting", "email" -> "Email". Style the badge small (text-[11px]).
