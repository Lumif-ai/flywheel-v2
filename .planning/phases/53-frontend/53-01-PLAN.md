Search and Filters bar:

Table:
- Use a simple HTML `<table>` with Tailwind classes (the app doesn't use a table library) — `w-full text-left` with `border-collapse`
- Columns: Name, Status, Fit Score, Contacts, Last Interaction, Next Action
- Column headers are clickable for sorting — clicking toggles `sort_by` and `sort_dir` params. Show a small arrow indicator for active sort. Sortable columns: `name`, `fit_score`, `last_interaction_at`, `next_action_due`
- Each row is clickable — uses `useNavigate` to navigate to `/accounts/${account.id}`
- Row hover state: subtle background highlight

Cell rendering:
- **Name**: bold text, domain below in caption style if present
- **Status**: `Badge` component with color mapping — prospect=blue, engaged=coral (#E94D35), customer=green, churned=gray, disqualified=red
- **Fit Score**: show tier as colored badge (excellent=green, good=blue, fair=amber, poor=gray) with numeric score next to it; show "—" if null
- **Contacts**: plain number
- **Last Interaction**: relative time format (e.g., "3d ago", "2h ago") using a simple formatter; "Never" if null
- **Next Action**: relative time + action type badge; "None" if null. Use `cardBorderColors.warning` color if overdue (date < now)

- Show "Showing X-Y of Z accounts" text
- Previous/Next buttons using `Button` component, disabled at boundaries
