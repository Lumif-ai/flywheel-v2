  <action>
Create `backend/src/flywheel/api/accounts.py` with `router = APIRouter(prefix="/accounts", tags=["accounts"])`.

Follow the exact patterns from `context.py`: Pydantic request/response models, `_paginated_response` helper, `get_tenant_db` and `require_tenant` dependencies.

**Pydantic models:**
- `AccountListItem` — id, name, domain, status, fit_score, fit_tier, contact_count (int), last_interaction_at, next_action_due, next_action_type, source
- `AccountDetail` — all AccountListItem fields plus intel (dict), contacts (list), recent_timeline (list of dicts with type discriminator — reuse for timeline later)
- `CreateAccountRequest` — name (required), domain, status, fit_score, fit_tier, intel, source (required)
- `UpdateAccountRequest` — all optional: name, domain, status, fit_score, fit_tier, intel, next_action_due, next_action_type
- `ContactResponse` — id, name, email, title, role_in_deal, linkedin_url, notes, source, created_at
- `CreateContactRequest` — name (required), email, title, role_in_deal, linkedin_url, notes, source (required)
- `UpdateContactRequest` — all optional: name, email, title, role_in_deal, linkedin_url, notes

**Endpoints (API-01):**

1. `GET /accounts/` — paginated list with query params:
   - `offset` (default 0), `limit` (default 50, max 100)
   - `status` filter (exact match, e.g. "prospect", "engaged")
   - `search` (ILIKE on name and domain)
   - `sort_by` (one of: name, fit_score, last_interaction_at, next_action_due, created_at; default: created_at)
   - `sort_dir` (asc/desc, default desc)
   - Query: select Account, add `.where()` for status/search filters, use `func.count()` subquery for contact_count via a correlated subquery or left join, order_by sort_by column, offset/limit. Return `_paginated_response`.

2. `GET /accounts/{account_id}` — full detail:
   - Fetch Account by id, 404 if not found
   - Eager-load contacts via `selectinload(Account.contacts)`
   - Fetch 10 most recent timeline entries: UNION of outreach_activities (type="outreach"), context_entries where account_id matches (type="context"), and uploaded_files where account_id matches via context_entries (type="document") — each with id, type, title/subject, date, summary/preview. Order by date DESC, limit 10.
   - For simplicity, use 3 separate queries (outreach, context_entries, uploaded_files linked via context_entries with account_id) and merge+sort in Python. This avoids complex UNION ALL with different column shapes.
   - Return AccountDetail shape.

3. `POST /accounts/` — create account:
   - Import `normalize_company_name` from `flywheel.utils.normalize`
   - Compute `normalized_name` from request name
   - Set `tenant_id` from user token
   - Flush and refresh, return 201 with account dict

4. `PATCH /accounts/{account_id}` — update account:
   - Fetch by id, 404 if not found
   - Apply only non-None fields from UpdateAccountRequest
   - If name is updated, recompute normalized_name
   - Set `updated_at = now()`
   - Commit, refresh, return updated account dict

**Endpoints (API-02):**

5. `GET /accounts/{account_id}/contacts` — list contacts for account:
   - Verify account exists (404 if not)
   - Select AccountContact where account_id matches, order by created_at
   - Return list (no pagination needed at contact scale)

6. `POST /accounts/{account_id}/contacts` — create contact:
   - Verify account exists (404)
   - Create AccountContact with tenant_id from user, account_id from path
   - Return 201

7. `PATCH /accounts/{account_id}/contacts/{contact_id}` — update contact:
   - Fetch contact by id, verify it belongs to account_id (404 if not found or mismatched)
   - Apply non-None fields, set updated_at
   - Return updated contact

8. `DELETE /accounts/{account_id}/contacts/{contact_id}` — delete contact:
   - Fetch contact, verify account_id match, 404 if not found
   - Hard delete (CASCADE is on the FK, but this is explicit single delete)
   - Return {"deleted": true, "id": str(contact_id)}

**Serialization helpers:**
- `_account_to_list_item(a: Account, contact_count: int) -> dict`
- `_account_to_detail(a: Account, contacts: list, timeline: list) -> dict`
- `_contact_to_dict(c: AccountContact) -> dict`

Use `from flywheel.utils.normalize import normalize_company_name` for POST/PATCH name normalization.
