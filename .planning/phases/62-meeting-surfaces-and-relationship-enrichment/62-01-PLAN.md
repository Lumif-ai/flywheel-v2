  <action>
Create the meetings feature slice data layer:

1. **types/meetings.ts** — Define TypeScript types matching backend response shapes exactly:
   - `Attendee` — `{ email: string | null; name: string | null; is_external: boolean }`
   - `MeetingSummary` — `{ tldr: string | null; key_decisions: string[]; action_items: Array<{ item: string; owner?: string; due?: string }>; pain_points: string[]; attendee_roles: Record<string, string>; meeting_type: string | null }`
   - `MeetingListItem` — id, title, meeting_date, duration_mins, attendees (Attendee[] | null), meeting_type, processing_status ('pending'|'processing'|'complete'|'failed'|'skipped'), account_id, summary (MeetingSummary | null), created_at
   - `MeetingDetail extends MeetingListItem` — adds skill_run_id, processed_at, updated_at, transcript_url? (optional), ai_summary? (optional)
   - `SyncResult` — `{ synced: number; skipped: number; already_seen: number; total_from_provider: number }`

2. **api.ts** — Follow `features/relationships/api.ts` pattern exactly:
   - `queryKeys.meetings` — `all: ['meetings']`, `list: (status?) => ['meetings', 'list', status ?? 'all']`, `detail: (id) => ['meetings', 'detail', id]`
   - `fetchMeetings(status?)` — GET /meetings/ with optional status param, limit 50
   - `fetchMeetingDetail(id)` — GET /meetings/{id}
   - `syncMeetings()` — POST /meetings/sync
   - `processMeeting(id)` — POST /meetings/{id}/process

3. **hooks/useMeetings.ts** — useQuery wrapping fetchMeetings, enabled when user is authenticated. Follow `useRelationships.ts` pattern.

4. **hooks/useMeetingDetail.ts** — useQuery wrapping fetchMeetingDetail(id), enabled when user + id exist.

5. **hooks/useSyncMeetings.ts** — useMutation wrapping syncMeetings(). On success: invalidate queryKeys.meetings.all, show toast.success with sync stats. On error: toast.error with message.

6. **hooks/useMeetingProcessing.ts** — Follow `useProfileRefresh.ts` pattern EXACTLY:
   - State: phase ('idle'|'processing'|'complete'|'error'), currentStage (string|null), sseUrl (string|null)
   - useSSE(sseUrl, handleEvent) from @/lib/sse.ts
   - handleEvent: 'stage' -> setCurrentStage, 'done' -> setPhase('complete') + setSseUrl(null) + invalidate detail + invalidate list + invalidate signals, 'error' -> setPhase('error') + setSseUrl(null)
   - startProcessing: setPhase('processing'), await processMeeting(meetingId), THEN setSseUrl to /api/v1/skills/runs/{run_id}/stream (only after await returns run_id)
   - Return { phase, currentStage, startProcessing }
  </action>
  <verify>Run `cd frontend && npx tsc --noEmit` — no type errors in the new files.</verify>
  <done>All 6 files exist with correct types, API functions, and hooks. TypeScript compiles cleanly.</done>
