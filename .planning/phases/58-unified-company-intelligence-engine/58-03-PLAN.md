  <name>Task 1: Create useProfileRefresh hook with startRefresh, startReset, and startFromRunId</name>
  <files>frontend/src/features/profile/hooks/useProfileRefresh.ts</files>
  <action>
Create a new hook file modeled on `useProfileCrawl.ts` but calling the new profile endpoints. Do NOT modify `useProfileCrawl` — create a separate hook per research recommendation (avoids God hook).

The hook has the same state shape as `useProfileCrawl`:
- `phase: 'idle' | 'refreshing' | 'complete' | 'error'`
- `crawlItems: CrawlItem[]` (reuse type from `@/features/onboarding/hooks/useOnboarding`)
- `crawlTotal: number`
- `crawlStatus: string | null`
- `error: { message: string; retryable: boolean } | null`

The SSE handler is identical to `useProfileCrawl`'s `handleEvent` — handles `stage`, `discovery` (not `text` — the skills/runs stream sends `discovery` events directly), `done`, `crawl_error`, `error` event types. For `discovery` events, map them to `CrawlItem`:
```typescript
case 'discovery': {
  const item: CrawlItem = {
    category: (event.data.category as string) ?? 'company_info',
    icon: (event.data.icon as string) ?? 'Building2',
    label: (event.data.label as string) ?? '',
    items: (event.data.items as string[]) ?? [],
    count: (event.data.count as number) ?? crawlItemsRef.current.length + 1,
  }
  setCrawlItems((prev) => [...prev, item])
  setCrawlTotal((prev) => prev + item.items.length)
  setCrawlStatus(null)
  break
}
```

Also handle `text` events the same way (in case the endpoint format changes or for backward compat).

Actions (all four must be implemented in this task):
- `startRefresh()`: resets state, POSTs to `/profile/refresh` via `api.post<{ run_id: string }>('/profile/refresh', {})`, sets `sseUrl` to `/api/v1/skills/runs/${res.run_id}/stream?token=${token}`. Get token from `useAuthStore.getState().token`. Set phase to `'refreshing'`.
- `startReset()`: same as `startRefresh()` but POSTs to `/profile/reset`. The response also has `run_id` and `deleted_count`.
- `startFromRunId(runId: string)`: sets phase to `'refreshing'`, resets all state (crawlItems, crawlTotal, crawlStatus, error), sets `sseUrl` to `/api/v1/skills/runs/${runId}/stream?token=${token}`. Does NOT make a POST call — the caller already has the run_id from their own POST. This is used by `DocumentAnalyzePanel` which does its own POST to `/profile/analyze-document` and passes the resulting run_id.
  ```typescript
  const startFromRunId = useCallback((runId: string) => {
    setPhase('refreshing')
    setCrawlItems([])
    setCrawlTotal(0)
    setCrawlStatus(null)
    setError(null)
    const token = useAuthStore.getState().token
    setSseUrl(`/api/v1/skills/runs/${runId}/stream?token=${token}`)
  }, [])
  ```
- `dismiss()`: resets everything back to `'idle'` (for closing the stream overlay after completion).

Use `useSSE(sseUrl, handleEvent)` from `@/lib/sse`.

On `done` event: `setPhase('complete')`, `queryClient.invalidateQueries({ queryKey: ['company-profile'] })`, `setSseUrl(null)`.

Export: `export function useProfileRefresh()` returning `{ phase, crawlItems, crawlTotal, crawlStatus, error, startRefresh, startReset, startFromRunId, dismiss }`.
