**EXTRACTION_SYSTEM_PROMPT** (multiline string):
```
You are an email intelligence extraction engine. Given an email's metadata and body,
extract structured intelligence into exactly these categories.

Respond ONLY with a valid JSON object using these exact keys:
- contacts: list of {name, title, company, email, role_in_context, notes, confidence}
- topics: list of {topic, relevance, context, confidence}
- deal_signals: list of {signal_type, description, confidence, counterparty}
- relationship_signals: list of {signal_type, description, people_involved, confidence}
- action_items: list of {action, owner, due_date, urgency, confidence}

Rules:
- confidence must be "high", "medium", or "low" for each item
- Use "high" when explicitly stated in the email (e.g., "I'm the VP of Sales")
- Use "medium" for strongly implied information
- Use "low" for inferred or uncertain information
- Use empty lists [] for categories with no data
- Only extract information explicitly stated or strongly implied — do not hallucinate
- relevance must be "high", "medium", or "low"
- urgency must be "high", "medium", or "low"
- due_date should be ISO format (YYYY-MM-DD) or null if not mentioned
```

**`_build_extraction_prompt(email: Email, body: str) -> tuple[str, str]`:**
- System prompt = EXTRACTION_SYSTEM_PROMPT
- User message = `f"Subject: {email.subject or '(no subject)'}\nFrom: {email.sender_name or email.sender_email}\nDate: {email.received_at.strftime('%Y-%m-%d')}\n\n{body[:8000]}"` -- truncate body to 8000 chars to stay within context window
- Return (system_prompt, user_message)

**`_parse_extraction_response(text: str) -> dict`:**
Follow the exact pattern from `email_scorer._parse_score_response`:
1. Try `json.loads(text)` first
2. If JSONDecodeError, try regex: `re.search(r'\{.*\}', text, re.DOTALL)` and parse the match
3. If still fails, return empty extraction: `{"contacts": [], "topics": [], "deal_signals": [], "relationship_signals": [], "action_items": []}`
4. Validate each key exists and is a list; default missing keys to `[]`
5. Validate confidence values on each item: if missing or not in ("high", "medium", "low"), default to "medium"
6. Log warning if fallback was used: `"_parse_extraction_response: used regex fallback for LLM output"`

Logic:
1. Initialize `results = {"created": 0, "incremented": 0, "total": 0}`
2. For each contact in `extracted["contacts"]`:
   - Call `write_contact(db, tenant_id, user_id, name=c["name"], title=c.get("title"), company=c.get("company"), email_address=c.get("email"), notes=c.get("notes"), source_label=_SOURCE_LABEL, confidence=c.get("confidence", "medium"), entry_date=email_date)`

3. For each topic in `extracted["topics"]`:
   - Call `write_insight(db, tenant_id, user_id, topic=t["topic"], relevance=t.get("relevance", "medium"), context_text=t.get("context", ""), source_label=_SOURCE_LABEL, confidence=t.get("confidence", "medium"), entry_date=email_date)`
4. For each deal signal in `extracted["deal_signals"]`:
   - Call `write_deal_signal(db, tenant_id, user_id, signal_type=d["signal_type"], description=d["description"], counterparty=d.get("counterparty"), source_label=_SOURCE_LABEL, confidence=d.get("confidence", "medium"), entry_date=email_date)`
5. For each relationship signal in `extracted["relationship_signals"]`:
   - Call `write_relationship_signal(db, tenant_id, user_id, signal_type=r["signal_type"], description=r["description"], people_involved=r.get("people_involved", []), source_label=_SOURCE_LABEL, confidence=r.get("confidence", "medium"), entry_date=email_date)`
6. For each action item in `extracted["action_items"]`:
   - Call `write_action_item(db, tenant_id, user_id, action=a["action"], owner=a.get("owner"), due_date_str=a.get("due_date"), urgency=a.get("urgency", "medium"), source_label=_SOURCE_LABEL, confidence=a.get("confidence", "medium"), entry_date=email_date)`
7. Wrap each writer call in try/except — log and continue on individual item failures (never lose other items because one fails)
8. Return results dict

**`extract_email_context(db, tenant_id, email, integration, api_key=None) -> dict | None`:**
Main entry point. Mirrors `score_email()` pattern.

Signature:
```python
async def extract_email_context(
    db: AsyncSession,
    tenant_id: UUID,
    email: Email,
    integration: Integration,
    api_key: str | None = None,
) -> dict | None:
```

Logic:
1. **Priority guard** (before any expensive work):
   ```python
   score_row = await db.execute(
       select(EmailScore.priority).where(EmailScore.email_id == email.id)
   )
   priority = score_row.scalar_one_or_none()
   if priority is None or priority < 3:
       logger.debug("extract_email_context: skipping email_id=%s priority=%s", email.id, priority)
       return None
   ```
2. Fetch body on-demand:
   ```python
   creds = await get_valid_credentials(integration)
   body = await get_message_body(creds, email.gmail_message_id)
   ```
3. If body is None or too short (< 50 chars stripped), try `email.snippet or ""`. If still < 20 chars, return None (nothing meaningful to extract).
4. Get model: `model = await get_engine_model(db, tenant_id, "context_extraction")`
5. Build prompt: `system_prompt, user_message = _build_extraction_prompt(email, body)`
6. Call Claude:
   ```python
   client = anthropic.AsyncAnthropic(api_key=api_key or settings.flywheel_subsidy_api_key)
   response = await client.messages.create(
       model=model,
       max_tokens=2000,
       system=system_prompt,
       messages=[{"role": "user", "content": user_message}],
   )
   text = response.content[0].text.strip()
   ```
7. **Explicitly discard body:** `del body` — comment: "PII posture: never retain email body beyond extraction"
8. Parse: `extracted = _parse_extraction_response(text)`
9. Write: `results = await _write_extracted_context(db, tenant_id, email.user_id, email.id, email.received_at.date(), extracted)`
10. Log summary (no PII): `"extract_email_context: email_id=%s tenant_id=%s created=%d incremented=%d total=%d", email.id, tenant_id, results["created"], results["incremented"], results["total"]`
11. Return `{"extracted": extracted, "results": results, "model": model}`

Wrap entire function body in try/except Exception:
- Log: `"extract_email_context failed email_id=%s tenant_id=%s: %s", email.id, tenant_id, exc`
- Return None

Do NOT call `db.commit()` — caller owns the transaction.
