**Imports:**

**Private helper `_write_entry()`:**
```python
async def _write_entry(
    db: AsyncSession,
    tenant_id: UUID,
    user_id: UUID,
    file_name: str,
    source: str,
    detail: str,
    content: str,
    confidence: str = "medium",
    entry_date: date | None = None,
    account_id: UUID | None = None,
) -> str:
```

Logic:
1. Truncate `content` to `_MAX_CONTENT_LENGTH` chars.
2. If `entry_date` is None, use `date.today()`.
3. Dedup query: `SELECT id, evidence_count FROM context_entries WHERE file_name=X AND source=X AND detail=X AND tenant_id=X AND date=X AND deleted_at IS NULL LIMIT 1`
4. If row exists: `UPDATE context_entries SET evidence_count = evidence_count + 1 WHERE id = row.id`. Return `"incremented"`.
5. If no row: `session.add(ContextEntry(...))` with all fields. Return `"created"`.
6. After creating a new entry, upsert ContextCatalog using `pg_insert(ContextCatalog).values(tenant_id=tenant_id, file_name=file_name, description=_CATALOG_DESCRIPTIONS.get(file_name, file_name), status="active").on_conflict_do_update(index_elements=["tenant_id", "file_name"], set_={"status": "active"})`.
7. Log at debug level: `"context_store_writer: %s entry file=%s detail=%s tenant=%s"` with the result, file_name, detail (truncated to 50 chars), and tenant_id. Never log content.

**Catalog descriptions dict:**
```python
_CATALOG_DESCRIPTIONS: dict[str, str] = {
    "contacts": "People and contacts extracted from emails and meetings",
    "insights": "Business insights, topics, and relationship signals",
    "action-items": "Action items and commitments extracted from communications",
    "deal-signals": "Deal and commercial signals from conversations",
}
```

Do NOT call `db.commit()` anywhere in this file.

Add four public async functions to `context_store_writer.py`, each building a structured detail tag and content string, then delegating to `_write_entry()`.

**`write_contact(db, tenant_id, user_id, name, title, company, email_address, notes, source_label, confidence, entry_date, account_id)`:**
- `detail_tag = f"contact:{name.lower().strip()}"` -- if company, append `f":{company.lower().strip()}"`
- Content lines: `Name: {name}`, then optionally `Title: {title}`, `Company: {company}`, `Email: {email_address}`, `Notes: {notes}` (skip None fields)
- `file_name = "contacts"`
- Pass `source=source_label` (caller provides, e.g., `"email-context-engine"`)

**`write_insight(db, tenant_id, user_id, topic, relevance, context_text, source_label, confidence, entry_date, account_id)`:**
- `detail_tag = f"insight:{topic.lower().strip()[:80]}"` -- truncate topic to 80 chars for tag stability
- Content: `Topic: {topic}\nRelevance: {relevance}\n{context_text}`
- `file_name = "insights"`

**`write_action_item(db, tenant_id, user_id, action, owner, due_date_str, urgency, source_label, confidence, entry_date, account_id)`:**
- `detail_tag = f"action:{action.lower().strip()[:80]}"`
- Content lines: `Action: {action}`, then optionally `Owner: {owner}`, `Due: {due_date_str}`, `Urgency: {urgency}`
- `file_name = "action-items"`

**`write_deal_signal(db, tenant_id, user_id, signal_type, description, counterparty, source_label, confidence, entry_date, account_id)`:**
- `detail_tag = f"deal:{signal_type.lower().strip()}:{(counterparty or 'unknown').lower().strip()[:40]}"`
- Content lines: `Signal: {signal_type}`, `Description: {description}`, optionally `Counterparty: {counterparty}`
- `file_name = "deal-signals"`

All four functions:
- Accept `source_label: str` (not hardcoded -- lets email engine pass `"email-context-engine"` and meeting processor could pass `"ctx-meeting-processor"`)
- Accept optional `confidence: str = "medium"`, `entry_date: date | None = None`, `account_id: UUID | None = None`
- Return the string result from `_write_entry()` (`"created"`, `"incremented"`)
- Have clear docstrings explaining the detail tag format and dedup behavior

Also add a convenience function:
**`write_relationship_signal(db, tenant_id, user_id, signal_type, description, people_involved, source_label, confidence, entry_date, account_id)`:**
- `detail_tag = f"relationship:{signal_type.lower().strip()}:{':'.join(sorted(p.lower().strip() for p in (people_involved or [])[:3]))}"`
- Content: `Signal: {signal_type}\nDescription: {description}\nPeople: {', '.join(people_involved or [])}`
- `file_name = "insights"` (relationship signals go to insights file, same as meeting processor pattern)
