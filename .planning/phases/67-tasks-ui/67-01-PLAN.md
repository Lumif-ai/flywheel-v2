  <action>
  Create the API module and all 6 React Query hooks:

  **api.ts:**
  - `queryKeys` factory object with: `all`, `list(filters?)`, `summary`, `detail(id)` — following meetings/api.ts pattern
  - API functions: `fetchTasks`, `fetchTaskSummary`, `fetchTask`, `createTask`, `updateTask`, `updateTaskStatus` — all using `api` from `@/lib/api`
  - Import types from `./types/tasks`

  **hooks/useTasks.ts:**
  - `useTasks(filters?: TaskFilters)` — `useQuery` with `queryKeys.tasks.list(filters)`, staleTime 30_000 (30s, tasks change frequently during triage)
  - Return typed `TasksListResponse`

  **hooks/useTaskSummary.ts:**
  - `useTaskSummary()` — `useQuery` with `queryKeys.tasks.summary`, staleTime 30_000

  **hooks/useTask.ts:**
  - `useTask(taskId: string | null)` — `useQuery` with `queryKeys.tasks.detail(taskId!)`, enabled only when taskId is truthy

  **hooks/useCreateTask.ts:**
  - `useCreateTask()` — `useMutation` calling `createTask`, invalidates `queryKeys.tasks.all` and `queryKeys.tasks.summary` on settled
  - Shows `toast.success('Task created')` on success, `toast.error(...)` on error

  **hooks/useUpdateTask.ts:**
  - `useUpdateTask()` — `useMutation` calling `updateTask(id, body)`, invalidates all task queries on settled
  - No toast on success (inline edits are frequent), `toast.error(...)` on error

  **hooks/useUpdateTaskStatus.ts (most critical):**
  - `useUpdateTaskStatus()` — `useMutation` with OPTIMISTIC UPDATES:
    - `onMutate`: cancel outgoing queries, snapshot previous data, optimistically remove task from current filtered list
    - `onError`: rollback to snapshot, show `toast.error('Could not update task. Please try again.')`
    - `onSettled`: invalidate `queryKeys.tasks.all` and `queryKeys.tasks.summary`
  - Follow the pattern from `useGraduate.ts` for optimistic update structure
  - Return mutation result so callers can use `onSuccess` for undo toasts

  All hooks go in `frontend/src/features/tasks/hooks/`. Follow existing codebase patterns: `useQueryClient` for cache operations, `toast` from `sonner` for notifications, `api` from `@/lib/api` for requests.
  </action>
