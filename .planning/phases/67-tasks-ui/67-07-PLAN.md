  <files>

  <action>
  Build the skill execution hook and wire it into the detail panel:

  **useSkillExecution.ts:**
  - `useSkillExecution()` returns a mutation-like interface: `{ execute, isExecuting, result, error }`
  - `execute(skillName: string, skillContext: Record<string, unknown>)`:
    1. POST to `/api/v1/skills/runs` with `{ skill_name: skillName, input_text: JSON.stringify(skillContext) }` using the `api` utility
    2. Receive `{ run_id, status, stream_url }` response
    3. If the codebase has an existing SSE pattern (check useMeetingProcessing or similar), follow it. Otherwise, use a simple fetch-based approach or just poll the run status
    4. Set `isExecuting: true` during execution
    5. On completion: set `result` with the output data, set `isExecuting: false`
    6. On error: set `error`, show `toast.error('Skill execution failed')`, set `isExecuting: false`
  - Use React state (useState) for `isExecuting`, `result`, `error` — this is ephemeral UI state within the panel lifecycle
  - NOTE: The SSE endpoint requires auth. Check if the existing `api` utility supports SSE or if there's a `fetchSSE` helper. If not, fall back to polling `GET /skills/runs/{run_id}` every 2 seconds until status is 'completed' or 'failed'. Keep it simple.

  **TaskDetailPanel.tsx updates:**
  - In the Skill Execution section (conditional on `task.suggested_skill`):
    - Remove the disabled "Coming soon" placeholder
    - Wire "Generate Deliverable" button to `execute(task.suggested_skill, task.skill_context || {})`
    - Button shows only when status is 'confirmed' or 'in_progress'
    - During execution: button shows spinner (use `Loader2` icon with `animate-spin` class) + "Generating..." text
    - On success: show output section with a link (use result data). If the result contains a file path or URL, link to it. Toast: "Deliverable generated"
    - On error: show error message inline + "Retry" button that re-triggers execute
    - If task.metadata already has generated output (from prior execution), show that with "Regenerate" option
  </action>

  **Search (TASK-15):**
  - In TasksPage header, add search input to the right of "+ Add" button
  - Input: placeholder "Search tasks...", with Search icon (Lucide) on left, X clear button on right when non-empty
  - State: `searchQuery` string state in TasksPage, debounced to 300ms using a simple `useEffect` + `setTimeout` pattern (no new deps)
  - Pass `debouncedSearch` down to all section components as a `searchFilter?: string` prop

  - When filtered results are empty in a section, show "No tasks matching '[query]'" text inline (not the full EmptyState)
  - Clear button (X) resets searchQuery to empty string

  **Staggered Entrance Animations (TASK-16):**
  - Use the existing `staggerDelay` function from `@/lib/animations` and `animationClasses.fadeSlideUp` CSS class
  - In TriageInbox, MyCommitments, PromisesToMe: wrap each card/row in a div with:
    ```
    className={animationClasses.fadeSlideUp}
    style={{ animationDelay: staggerDelay(index) }}
    ```
  - This applies 50ms stagger between cards on initial render
  - Add `@media (prefers-reduced-motion: reduce)` rule in index.css:
    ```
    @media (prefers-reduced-motion: reduce) {

        animation: none !important;
        opacity: 1 !important;
        transform: none !important;
      }
    }
    ```
