  <action>
  Build the complete Focus Mode component:

  **focusModeStore.ts (Zustand store for ephemeral UI state):**
  - State: `currentIndex: number`, `animationDirection: 'left' | 'right' | 'down' | null`, `isEditing: boolean`, `isComplete: boolean`
  - Actions: `nextTask(direction)`, `setEditing(bool)`, `setComplete()`, `reset()`
  - This is UI-only state — task data comes from React Query, status changes go through hooks

  **FocusMode.tsx (organism):**
  - Props: `tasks: Task[]` (triage tasks passed from TriageInbox), `isOpen: boolean`, `onClose: () => void`
  - Renders as a fixed full-viewport overlay when `isOpen` is true
  - Backdrop: semi-transparent `var(--bg-overlay)` or `rgba(0,0,0,0.5)`

  **Layout (centered, max-width 560px):**

  1. **Header bar** (fixed at top of overlay):
     - "Reviewing N of M" text (14px/500)
     - Exit button (X icon) top-right, calls onClose
     - Progress bar below: 4px height, full-width, track `var(--subtle-border)`, fill `var(--brand-coral)`, width = `(reviewed / total) * 100%`, CSS transition on width

  2. **Card area** (centered vertically):
     - Card: max-width 560px, bg `var(--card-bg)`, border-radius 16px, shadow `--shadow-lg`, padding 32px
     - Content (current task):
       - Title: 20px/600 in `var(--heading-text)`
       - Divider line
       - Meeting: label "Meeting" + meeting name + date (14px/400)
       - Account: label "Account" + account name (14px/400) — omit row if no account
       - Commitment direction badge
       - Suggested skill chip (TaskSkillChip) if exists
       - Description/context if exists (14px/400, `var(--secondary-text)`)
       - Priority badge + Due date
     - **Edit mode** (when `isEditing`): replace title with input, add priority toggle and date picker inline. Enter/Escape exits edit mode.

  3. **Action bar** (below card, centered):
     - Three buttons in a row, gap 16px:
       - Dismiss: outline variant, icon X, label "Dismiss", hint "< key" below in 12px/400 `var(--secondary-text)`. Hover: red tint
       - Later: outline variant, icon Clock, label "Later", hint "v key". Hover: amber tint
       - Confirm: default (coral) variant, icon CheckCircle, label "Confirm", hint "> key". Slightly larger than others (primary action)
     - Below action bar: "Edit before confirming" link (14px/400, `var(--brand-coral)`) — sets `isEditing: true`

  **Card transition animations (CSS transitions/keyframes):**
  - Confirm exit: translateX(200px) + rotate(3deg) + opacity(0), 250ms ease-in
  - Dismiss exit: translateX(-200px) + rotate(-3deg) + opacity(0), 250ms ease-in
  - Later exit: translateY(50px) + opacity(0), 250ms ease-in
  - New card enter: from translateY(30px) + opacity(0) to translateY(0) + opacity(1), 300ms cubic-bezier(0.2, 0, 0, 1) with 100ms delay after exit
  - Implementation: use CSS classes toggled by state. Set `animationDirection` in store, apply CSS class, wait for animation end, then advance index.
  - `prefers-reduced-motion`: instant swap, no translate/rotate

  **Keyboard shortcuts** (useEffect on document keydown, only when isOpen and NOT isEditing):
  - `ArrowRight` or `Enter`: confirm current task
  - `ArrowLeft` or `Backspace`: dismiss current task
  - `ArrowDown` or `s`/`S`: save for later
  - `e`/`E`: toggle edit mode
  - `Escape`: exit focus mode (onClose)
  - Call `preventDefault()` on handled keys to prevent browser scrolling

  **Action flow:**
  1. User presses key or clicks button
  2. Set `animationDirection` to trigger exit animation
  3. Call `useUpdateTaskStatus` with appropriate status
  4. Show undo toast (same pattern as list triage)
  5. After 250ms (exit animation), advance `currentIndex`
  6. Clear `animationDirection` to trigger enter animation for next card
  7. If `currentIndex >= tasks.length`, show completion state

  **Completion state:**
  - Large CheckCircle icon (48px) in `var(--success)` with scale-up CSS animation
  - "All caught up" in section title typography
  - "[N] tasks reviewed" in caption
  - Auto-close after 2 seconds via setTimeout (with "Close" button for immediate exit)
  - Clear timeout on unmount

  **Accessibility:**
  - `role="dialog"`, `aria-modal="true"`, `aria-label="Task review"`
  - Focus trap: trap focus within overlay using `tabIndex` management or a focus trap utility. At minimum, auto-focus the card area on open and prevent Tab from escaping.
  - `aria-live="polite"` region: announce "Reviewing task N of M" on each transition
  - Screen reader only text for progress bar percentage

  **TriageInbox.tsx updates:**
  - Wire "Review All" button: `onClick={() => setFocusModeOpen(true)}`
  - Render `<FocusMode tasks={triageTasks} isOpen={focusModeOpen} onClose={() => setFocusModeOpen(false)} />`
  - Hide "Review All" button when triage list is empty
