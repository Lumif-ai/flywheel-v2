  Build the slide-in detail panel using the existing Sheet component:

  **TaskDetailPanel.tsx (organism):**
  - Uses `Sheet` component with `side="right"`, override width: `className="sm:max-w-[480px] w-full"`
  - Props: `taskId: string | null`, `onClose: () => void` — when taskId is non-null, panel opens
  - Fetches task data using `useTask(taskId)` hook
  - Uses `useUpdateTask()` for field edits and `useUpdateTaskStatus()` for status changes

  **Panel sections (top to bottom):**

  1. **Header:**
     - Task title (18px/600), editable on click — click transforms to input, Enter/blur saves via `useUpdateTask`, Escape reverts
     - Close button (X icon) top-right, calls `onClose`
     - SheetTitle for accessibility

  2. **Metadata grid** (alternating row backgrounds `var(--page-bg)` / `var(--card-bg)`):
     - Status: `TaskStatusBadge` + dropdown select showing only valid transitions from current status (read from `VALID_TRANSITIONS[task.status]`). On change, call `useUpdateTaskStatus`. Use a simple `<select>` styled with Tailwind or a DropdownMenu.
     - Priority: three-option toggle (high/medium/low buttons), selected state highlighted, updates via `useUpdateTask` immediately
     - Due date: display formatted date + Calendar icon. On click, show `<input type="date">` for native date picker. Updates via `useUpdateTask` immediately.
     - Account: display account name as link to `/accounts/${task.account_id}` (read-only, no edit per spec gap #2)
     - Source meeting: display meeting name as link to meeting detail (read-only)
     - Task type: display label (read-only)
     - Commitment direction: display label (read-only)

  3. **Description:**
     - Textarea, editable, auto-save on blur with 500ms debounce using `useUpdateTask`
     - Placeholder "Add a description..." when empty
     - Min-height 80px, resize-y

  4. **Skill Execution (conditional — only if task.suggested_skill):**
     - Show `TaskSkillChip` with skill name
     - Show context preview: JSON.stringify(task.skill_context) truncated to 100 chars if exists
     - "Generate Deliverable" Button (variant default, with Zap icon) — placeholder onClick for now (wired in plan 07 TASK-14). Disabled with tooltip "Coming soon" for now.
     - If task.metadata has generated output, show "Generated Output" section with output link

  5. **Actions footer** (sticky at bottom, border-top):
     - "Mark Complete" Button (variant default, primary) — calls `useUpdateTaskStatus(task.id, 'done')`, closes panel, shows toast "Task completed"
     - Disabled if current status doesn't allow `done` transition (check `VALID_TRANSITIONS`)
     - "Dismiss" Button (variant ghost, destructive text) — calls `useUpdateTaskStatus(task.id, 'dismissed')`, closes panel, shows toast "Task dismissed"

  **Behaviors:**
  - Panel open: Sheet handles 300ms slide animation
  - Panel close: Escape key (Sheet handles), close button, onOpenChange
  - Focus trapped within panel when open on mobile (Sheet handles)
  - Preserve main content scroll position (Sheet handles overlay behavior)
  - All field changes save immediately (no Save button) with `toast.error` on failure
