  <action>
  Build the Vim-inspired keyboard navigation for the Tasks page:

  **useTaskKeyboardNav.ts (hook):**
  - Props/params: `options: { enabled: boolean, onSelect: (taskId: string) => void }`
  - Maintains a `focusedIndex` state (number, starts at -1 = nothing focused)
  - Collects all task card elements on the page using `data-task-id` attribute (set on each card component)
  - `useEffect` on document keydown (only when `enabled` is true):
    - `j`: increment focusedIndex (move to next card), scroll into view, apply focus ring
    - `k`: decrement focusedIndex (move to previous card)
    - `Enter` or `Space`: call `onSelect(currentTaskId)` to open detail panel
    - Do NOT wrap: j on last card does nothing, k on first card does nothing
  - **Disabled when:** an `<input>`, `<textarea>`, or `<select>` is the active element (check `document.activeElement?.tagName`), OR when enabled=false
  - Focus ring: add/remove a CSS class (`task-card-focused`) that applies `outline: 2px solid var(--brand-coral); outline-offset: 2px`
  - Clean up: remove keydown listener and focus ring class on unmount
  - Re-calculate card list when task data changes (use a MutationObserver or re-query on each keypress)

  **TasksPage.tsx updates:**
  - Add `data-task-id={task.id}` attribute to each TaskTriageCard, TaskCommitmentCard, and TaskWatchlistItem wrapper
  - Call `useTaskKeyboardNav({ enabled: !selectedTaskId && !showQuickAdd && !focusModeOpen, onSelect: (id) => setSelectedTaskId(id) })`
  - The `enabled` flag disables page shortcuts when detail panel, quick-add, or focus mode is open
  - Pass the data-task-id requirement down to card components as needed

  **Card component updates:**
  - In TaskTriageCard, TaskCommitmentCard, TaskWatchlistItem: ensure the outermost wrapper element has `data-task-id={task.id}` attribute
  - Add CSS for `.task-card-focused`: `outline: 2px solid var(--brand-coral); outline-offset: 2px;` (add to index.css or as inline style via the hook)
  </action>

  </verify>

  - Shortcuts disabled when typing in inputs or when panel/focus mode is open

  - No shortcut conflicts with focus mode or detail panel
