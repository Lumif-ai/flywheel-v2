Create `backend/src/flywheel/api/relationships.py` following the outreach.py pattern:

**Router setup:**
- `router = APIRouter(tags=["relationships"])` — no prefix, full paths explicit
- Import deps: `get_tenant_db`, `require_tenant`, `TokenPayload`, models

**Pydantic schemas:**
- `RelationshipListItem`: id, name, domain, relationship_type (list[str]), entity_level, relationship_status, ai_summary (str|None), signal_count (int), primary_contact_name (str|None), last_interaction_at, created_at
- `RelationshipDetail`: extends ListItem with contacts (list), recent_timeline (list of last 10 ContextEntries), ai_summary, ai_summary_updated_at, commitments (list — future, empty for now)
- `UpdateTypeRequest`: types (list[str]) — Pydantic validator enforces min length 1, all values in {"prospect", "customer", "advisor", "investor"}
- `GraduateRequest`: types (list[str]) — same validation, plus optional entity_level (default "company")

**ALLOWED_TYPES constant:** `{"prospect", "customer", "advisor", "investor"}`

**Endpoints:**

1. `GET /relationships/` (RAPI-01):
   - Query params: `type` (optional str, filter), `offset` (int=0), `limit` (int=50)
   - Base query: `select(Account).where(Account.tenant_id == user.tenant_id, Account.graduated_at.isnot(None))`
   - CRITICAL: The `graduated_at.isnot(None)` predicate MUST always be present — this is the partition predicate
   - If `type` param provided and in ALLOWED_TYPES: add `.where(Account.relationship_type.any(type))`
   - Use correlated subquery for primary_contact (same pattern as accounts.py lines 225-232: `select(AccountContact.name).where(AccountContact.account_id == Account.id).correlate(Account).order_by(AccountContact.created_at.asc()).limit(1).scalar_subquery()`)
   - Use correlated subquery for signal_count: count of ContextEntries where account_id matches and deleted_at IS NULL
   - Order by `updated_at desc`, apply offset/limit
   - Return `{"items": [...], "total": count}`

2. `GET /relationships/{id}` (RAPI-02):
   - Fetch account with `Account.id == id, Account.tenant_id == user.tenant_id, Account.graduated_at.isnot(None)`
   - 404 if not found (or if not graduated — same effect)
   - Eagerly load contacts via `selectinload(Account.contacts)`
   - Fetch recent timeline: `select(ContextEntry).where(ContextEntry.account_id == id, ContextEntry.deleted_at.is_(None)).order_by(ContextEntry.date.desc()).limit(10)`
   - Return ai_summary from column (may be NULL) — NEVER call LLM here
   - Serialize contacts as list of dicts (id, name, title, email, linkedin_url, role, created_at)

3. `PATCH /relationships/{id}/type` (RAPI-03):
   - Validate request body: `types` must be non-empty list, all values in ALLOWED_TYPES
   - If empty list: return 422 with `{"detail": "At least one relationship type required"}`
   - If unknown values: return 422 with `{"detail": "Unknown type: {value}. Allowed: prospect, customer, advisor, investor"}`
   - Fetch account with graduated_at partition predicate, 404 if not found
   - Update `account.relationship_type = request.types` and `account.updated_at = now`
   - Return updated account

4. `POST /relationships/{id}/graduate` (RAPI-04):
   - Fetch account by id + tenant_id (no graduated_at predicate here — we're graduating a non-graduated account)
   - If account.graduated_at is not None: return 409 "Account already graduated"
   - Set `account.graduated_at = now`, `account.relationship_type = request.types`, `account.entity_level = request.entity_level or "company"`, `account.updated_at = now`
   - Log ContextEntry (same pattern as _graduate_account in outreach.py)
   - Commit and return the account

**Register in main.py:**
- Add import: `from flywheel.api.relationships import router as relationships_router`
- Add: `app.include_router(relationships_router, prefix="/api/v1")` after the accounts_router line
