Create `backend/src/flywheel/services/synthesis_engine.py` following the onboarding_streams.py LLM pattern:

**Imports:**
- `anthropic` (conditional import like onboarding_streams.py), `datetime`, `timezone`, `timedelta`, `UUID`
- `from flywheel.config import settings`
- `from flywheel.services.circuit_breaker import anthropic_breaker`
- `from flywheel.db.models import Account, ContextEntry`
- `from sqlalchemy.ext.asyncio import AsyncSession`
- `from sqlalchemy import select, func`
- `from fastapi import HTTPException`

**Constants:**
- `_HAIKU_MODEL = "claude-haiku-4-5-20251001"` (same as onboarding_streams.py)
- `_RATE_LIMIT_WINDOW = timedelta(minutes=5)`
- `_MIN_CONTEXT_ENTRIES = 3` (for ask endpoint guard)

**Class SynthesisEngine** (stateless, all methods take db session + account):

1. `@staticmethod async def enforce_rate_limit(account: Account) -> None`:
   - If `account.ai_summary_updated_at` is None: return (never synthesized, allow)
   - `window = datetime.now(timezone.utc) - _RATE_LIMIT_WINDOW`
   - If `account.ai_summary_updated_at > window`: compute `retry_after` seconds, raise `HTTPException(status_code=429, detail={"error": "SynthesisRateLimitExceeded", "message": "AI synthesis is rate-limited to once per 5 minutes.", "code": 429}, headers={"Retry-After": str(max(retry_after, 1))})`

2. `@staticmethod async def generate(db: AsyncSession, account: Account) -> str | None`:
   - Fetch context entries: `select(ContextEntry).where(ContextEntry.account_id == account.id, ContextEntry.deleted_at.is_(None)).order_by(ContextEntry.date.desc()).limit(20)`
   - If fewer than _MIN_CONTEXT_ENTRIES entries: set `account.ai_summary = None`, `account.ai_summary_updated_at = datetime.now(timezone.utc)`, return None (graceful degradation for sparse data — still updates timestamp so rate limit applies)
   - Build system prompt: "You are an AI assistant summarizing a business relationship. Provide a concise 2-3 paragraph summary covering: relationship history, key interactions, current status, and recommended next steps."
   - Build user content: concatenate context entries as structured text (date, source, content)
   - Call LLM using circuit breaker pattern from onboarding_streams.py:
     ```python
     if not anthropic_breaker.can_execute():
         raise HTTPException(status_code=503, detail="AI service temporarily unavailable")
     try:
         client = anthropic.AsyncAnthropic(api_key=settings.flywheel_subsidy_api_key)
         response = await client.messages.create(
             model=_HAIKU_MODEL, max_tokens=2048,
             system=system_prompt,
             messages=[{"role": "user", "content": user_content}],
         )
         anthropic_breaker.record_success()
         summary = response.content[0].text.strip()
     except Exception:
         anthropic_breaker.record_failure()
         raise
     ```
   - Update account: `account.ai_summary = summary`, `account.ai_summary_updated_at = datetime.now(timezone.utc)`, `account.updated_at = datetime.now(timezone.utc)`
   - Return summary

3. `@staticmethod async def ask(db: AsyncSession, account: Account, question: str) -> dict`:
   - Fetch context entries (same query as generate, but all non-deleted entries up to 50)
   - Count entries: if fewer than _MIN_CONTEXT_ENTRIES, return `{"answer": "Not enough context to answer questions about this relationship. Add more notes or interactions first.", "sources": [], "insufficient_context": True}` — do NOT call LLM
   - Build system prompt: "You are answering questions about a business relationship. Use ONLY the context provided. When citing information, include [source:{entry_id}] markers referencing the context entry UUID that contains the information."
   - Build user content: each context entry formatted as `[ID: {entry.id}] [{entry.date}] [{entry.source}] {entry.content}` followed by the question
   - Call LLM (same circuit breaker pattern as generate)
   - Parse response: extract `[source:UUID]` markers from response text using regex
   - Return `{"answer": response_text, "sources": [{"id": str(entry_id), "source": entry.source, "date": str(entry.date)} for matched entries], "insufficient_context": False}`

Add two endpoints to the existing relationships router created in Plan 01:

**Pydantic schemas (add to existing file):**
- `SynthesizeResponse`: ai_summary (str|None), ai_summary_updated_at (datetime|None), insufficient_context (bool, default False)
- `AskRequest`: question (str) — Pydantic validator: min length 5, max length 1000
- `AskResponse`: answer (str), sources (list[dict]), insufficient_context (bool)

**Import:** `from flywheel.services.synthesis_engine import SynthesisEngine`

**Endpoints:**

1. `POST /relationships/{id}/synthesize` (RAPI-07):
   - Fetch account with `Account.id == id, Account.tenant_id == user.tenant_id, Account.graduated_at.isnot(None)` — 404 if not found
   - CRITICAL ORDER — this is the most important detail in the entire plan:
     a. FIRST: call `SynthesisEngine.enforce_rate_limit(account)` — raises 429 if within window
     b. SECOND: call `summary = await SynthesisEngine.generate(db, account)`
     c. The rate limit check MUST run BEFORE generate(), even if ai_summary is currently NULL
   - `await db.commit()`
   - `await db.refresh(account)`
   - Return `SynthesizeResponse(ai_summary=account.ai_summary, ai_summary_updated_at=account.ai_summary_updated_at)`

   Why this order matters: Success criterion 2 says "called twice within 5 minutes returns 429 on the second call" and "called with null ai_summary returns cached null, not a new LLM invocation." If generate() runs first and sees NULL, it might return early without triggering 429. The rate limit gate must come first.

2. `POST /relationships/{id}/ask` (RAPI-08):
   - Fetch account with partition predicate — 404 if not found
   - Validate request body (AskRequest)
   - Call `result = await SynthesisEngine.ask(db, account, request.question)`
   - Return `AskResponse(**result)`
   - Note: No rate limit on ask — only synthesize is rate-limited
