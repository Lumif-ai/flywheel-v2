Create `backend/src/flywheel/api/signals.py`:

**Router setup:**
- `router = APIRouter(tags=["signals"])` — no prefix
- Import deps: `get_tenant_db`, `require_tenant`, models, `func`, `select`, `and_`, `case`

**Signal taxonomy constants (SIG-02):**
```python
SIGNAL_TYPES = {
    "reply_received": {"priority": 1, "description": "New reply received"},
    "followup_overdue": {"priority": 2, "description": "Follow-up is overdue"},
    "commitment_due": {"priority": 2, "description": "Commitment coming due"},
    "stale_relationship": {"priority": 3, "description": "No interaction in 90+ days"},
}
```

**Pydantic schemas:**
- `SignalTypeCounts`: reply_received (int), followup_overdue (int), commitment_due (int), stale_relationship (int)
- `TypeBadge`: type (str), label (str), total_signals (int), counts (SignalTypeCounts)
- `SignalsResponse`: types (list[TypeBadge]), total (int)

**Helper: `_compute_signals_for_type()`:**
- Takes: db, tenant_id, type_value (str), now (datetime)
- Base filter: `Account.tenant_id == tenant_id, Account.graduated_at.isnot(None), Account.relationship_type.any(type_value)`
- CRITICAL: `graduated_at.isnot(None)` must be present — signals exclude pipeline-only accounts
- Compute counts for each signal type:
  - `reply_received`: accounts where last outreach activity status == "replied" and created_at > 7 days ago (recent replies). Query: join OutreachActivity, filter status="replied", created_at > now - 7 days, count distinct account_id
  - `followup_overdue`: accounts where `next_action_due IS NOT NULL AND next_action_due < now`. Simple count query on Account table
  - `commitment_due`: accounts where `next_action_type = 'commitment' AND next_action_due IS NOT NULL AND next_action_due < now + 7 days`. Count query (commitments due within a week)
  - `stale_relationship`: accounts where `last_interaction_at IS NOT NULL AND last_interaction_at < now - 90 days` OR `last_interaction_at IS NULL AND created_at < now - 90 days`. Count query
- Return SignalTypeCounts with all four counts

**Endpoint:**

`GET /signals/` (SIG-01):
- Compute signals for each of the four relationship types: prospect, customer, advisor, investor
- Call `_compute_signals_for_type()` four times (one per type) — simple separate queries, not one complex window function
- Build TypeBadge for each type: sum all signal counts for total_signals
- Return `SignalsResponse(types=[...], total=sum_of_all)`

**Register in main.py:**
- Add import: `from flywheel.api.signals import router as signals_router`
- Add: `app.include_router(signals_router, prefix="/api/v1")` after the relationships_router line
