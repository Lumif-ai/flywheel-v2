Create `backend/src/flywheel/seed_crm.py` following the exact same CLI pattern as `migration_tool.py` (argparse, asyncio.run, create_async_engine, async_sessionmaker).

**CLI arguments:**
- `--tenant-id` (required) — target tenant UUID
- `--gtm-dir` (default `~/.claude/gtm-stack/`) — directory containing xlsx, outreach-tracker.csv, pipeline-runs.json
- `--downloads-dir` (default `~/Downloads/`) — directory containing scored CSV files referenced by pipeline-runs.json
- `--database-url` (optional, falls back to `flywheel.config.settings.database_url`)
- `--dry-run` — parse and report counts without inserting
- `--verbose` — detailed per-file output
- `--force` — delete existing seeded data for this tenant before re-inserting

**File parsing (4 sources):**

1. **gtm-leads-master.xlsx** — use openpyxl to read:
   - Sheet "Company Summary": Company, Industry, Best Score → Account rows (name, intel.industry, fit_score). Source: "gtm-leads-master"
   - Sheet "All Leads Scored": Name, Title, Company, Email, Score, Tier → AccountContact rows. Also extract fit_score/fit_tier for the Account if not already set from Company Summary.
   - Sheet "Outreach Candidates": Skip — subset of All Leads Scored, no unique data.

2. **outreach-tracker.csv** — use csv.DictReader:
   - Each row creates/updates an AccountContact (Name, Email, Company, Title)
   - If Email_Sent == "Yes" and Email_Sent_Date is set → create OutreachActivity with channel="email", direction="outbound", status="sent", sent_at=Email_Sent_Date
   - If LinkedIn_Status is set and not empty → create OutreachActivity with channel="linkedin", direction="outbound", status=LinkedIn_Status.lower(), sent_at=LinkedIn_Date (if set)
   - Notes column → contact.notes

3. **Scored CSVs** (from pipeline-runs.json) — use csv.DictReader:
   - Read pipeline-runs.json, iterate entries with status=="complete"
   - For each run, resolve csv_path (expand ~ to home dir)
   - If CSV exists, read it. Each row: Company → Account (enriched with Company_Website→domain, Company_Description→intel.description, Fit_Score→fit_score, Fit_Tier→fit_tier)
   - Each row: Name, Email, Title, LinkedIn_URL → AccountContact
   - Source for these: "pipeline-{run_id}"

4. **pipeline-runs.json** — already consumed above for CSV paths. No separate table rows needed.

**Normalization and deduplication:**

- Import `normalize_company_name` from `flywheel.utils.normalize`
- For every company name encountered, compute `normalized_name = normalize_company_name(raw_name)`
- Skip rows where normalized_name is empty (malformed company names)

**Upsert strategy (use `sqlalchemy.dialects.postgresql.insert` with `on_conflict_do_update`):**

- **Accounts:** `INSERT ... ON CONFLICT (tenant_id, normalized_name) DO UPDATE SET` — merge fit_score (keep higher), fit_tier (keep if new is better), domain (keep first non-null), intel (jsonb merge with `||`), source (append new source). The `name` column should keep the first (most complete) version.
- **AccountContacts:** Dedup by (account_id, email) when email is not null, or (account_id, name) when email is null. Use a two-step approach: first try to find existing contact by email within the account, then upsert. For contacts without email, check by normalized name to avoid exact duplicates. Use `on_conflict_do_nothing` or manual SELECT+INSERT since there's no unique constraint on contacts — do a SELECT first, INSERT if not found.
- **OutreachActivities:** Dedup by (account_id, channel, sent_at). Since there's no unique constraint, do SELECT first, INSERT if not found. Skip if sent_at is null.

**Processing order:**
1. Parse all files into intermediate data structures (dicts keyed by normalized_name)
2. Merge account data across sources (Company Summary + scored CSVs + outreach tracker companies)
3. Upsert all Accounts in one pass → collect account_id map (normalized_name → account row id)
4. Upsert all Contacts → collect contact_id map (account_id + email → contact row id)
5. Insert OutreachActivities linking to account_id and contact_id

**Summary output:** Print counts of accounts, contacts, activities created/updated/skipped, similar to migration_tool.py's `_print_summary`.

**Error handling:** Wrap each file parse in try/except, log errors, continue processing other files. If a scored CSV path from pipeline-runs.json doesn't exist, log a warning and skip it.

**Important implementation notes:**
- Use `from __future__ import annotations` at top
- The module docstring should show usage: `python -m flywheel.seed_crm --tenant-id <uuid> [--gtm-dir ...] [--dry-run] [--verbose]`
- Include `if __name__ == "__main__": sys.exit(main())` at bottom
- For date parsing from CSV (Email_Sent_Date, LinkedIn_Date), handle format "YYYY-MM-DD" and be lenient with empty strings
- For fit_score from xlsx, handle both int and float values; from scored CSVs the column is "Fit_Score" (string, parse to float)
- The intel JSONB field should accumulate: {"industry": "...", "description": "...", "fit_reasoning": "..."} from various sources
