    **TimelineTab.tsx:**
    Props: `{ timeline: TimelineItem[] }`
    - Each entry renders as a row in a vertical list with:
      - Left: icon based on source field (Mail for email, MessageSquare for note, Calendar for meeting, FileText for document — default Activity)
      - Direction indicator: if direction === 'inbound' show ArrowDownLeft icon in secondary text; if 'outbound' show ArrowUpRight. These values come from the backend which derives them from ContextEntry.source (see Plan 01 Task 1).
      - Contact name (if exists, muted text) — also derived server-side
      - Content/title (primary text, 1 line truncated)
      - Time ago (relative, right-aligned, secondary text)
    - Entries are clickable — `onClick` expands to show full content text below the summary row (local state toggle per entry)
    - Show max 10 entries initially; if timeline.length > 10, show "Show more" button that reveals all
    - Empty state: "No timeline entries yet" with Activity icon

    **PeopleTab.tsx:**
    Props: `{ contacts: ContactItem[] }`
    - Each contact renders as a BrandedCard (variant="info", no left border):
      - Avatar size="xl" (48px) with initials
      - Name (font-weight 600), title below (secondary text)
      - Email as mailto link
      - LinkedIn icon that opens linkedin_url in new tab (if exists)
      - Role badge using badge-translucent with inline coral styles
      - Date label: "Added X days ago" using `contact.created_at` (NOT last_contacted_at — backend ContactItem does not have that field). Use relative time formatting.
    - Cards laid out in a 2-column grid: `className="grid grid-cols-1 md:grid-cols-2 gap-4"`
    - Empty state: "No contacts found" with Users icon

    **IntelligenceTab.tsx:**
    Props: `{ intel: Record<string, unknown> }`
    - Display 6 labeled data points: Pain, Budget, Competition, Champion, Blocker, Fit Reasoning
    - For each label, look up matching key in intel object (case-insensitive match: check for "pain", "Pain", "budget", "Budget", etc.)
    - The intel data comes from the backend's Account.intel JSONB column, now exposed via GET /relationships/{id} (see Plan 01 Task 1).
    - Render as a 2-column grid of BrandedCards (variant="info"):
      - Label (font-weight 600, uppercase tracking)
      - Value text (or "Not yet captured" in muted italic if key not found)
    - Each field shows a small Edit2 (Pencil) icon on hover — clicking opens the field as an editable textarea. For this phase, editing shows a toast.info("Intelligence editing coming soon") stub. Do NOT wire to any API.
    - Empty state (all 6 fields missing): "No intelligence gathered yet. Add notes to build this profile." with Brain icon

    **CommitmentsTab.tsx:**
    Props: `{ commitments: unknown[] }`
    - Two-column layout: "What You Owe" (left) and "What They Owe" (right)
    - Column headers: font-weight 600, secondary text, with icon (ArrowUpRight for "You Owe", ArrowDownLeft for "They Owe")
    - Since backend returns `[]` for commitments (reserved field), render the two-column structure with empty state in each column: "No commitments tracked yet"
    - When commitments exist (future-proofing): each commitment would render with title, due date, source attribution, and overdue highlighting (if due date < now, render with `style={{ color: 'var(--error)' }}` and bold)
    - Overall empty state: render the two-column structure (not a single EmptyState) with placeholder text in each column
