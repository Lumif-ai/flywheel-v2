  <name>Task 1: AskPanel component with dual-mode input and source citations</name>
  <files>frontend/src/features/relationships/components/AskPanel.tsx</files>
  <action>
    Create `AskPanel.tsx` following the state machine pattern from 57-RESEARCH.md Pattern 5:

    Props: `{ accountId: string; aiSummary: string | null; aiSummaryUpdatedAt: string | null }`

    **State machine:**
    ```
    type PanelMode = 'idle' | 'asking' | 'saving_note' | 'synthesizing'
    const [mode, setMode] = useState<PanelMode>('idle')
    const [inputValue, setInputValue] = useState('')
    const [lastAnswer, setLastAnswer] = useState<AskResponse | null>(null)
    ```

    **Hooks:**
    - `const noteMutation = useCreateNote()` — for saving notes
    - `const askMutation = useAsk()` — for Q&A
    - `const synthesizeMutation = useSynthesize()` — for refreshing AI summary

    **Intent heuristic:**
    `const isQuestion = (text: string) => text.trim().endsWith('?')`

    **handleSubmit function:**
    - If input is empty, return
    - If isQuestion(input): setMode('asking'), call askMutation.mutate({ accountId, question: input })
      - onSuccess: setLastAnswer(data), setMode('idle')
      - onError: setMode('idle'), toast.error from sonner
    - Else: setMode('saving_note'), call noteMutation.mutate({ accountId, content: input })
      - onSuccess: setMode('idle'), toast.success('Note saved'), clear lastAnswer
      - onError: setMode('idle'), toast.error
    - Always clear inputValue after submit

    **Layout (top to bottom):**

    1. **Section header:** "AI Context" in sectionTitle typography, with a RefreshCw icon button for synthesis refresh
       - On click: call synthesizeMutation.mutate({ accountId })
       - While synthesizing: spin the RefreshCw icon (add `animate-spin` class)
       - 429 error handling is in useSynthesize hook already (toast)

    2. **AI Summary section:**
       - If aiSummary is not null: render in a div with muted background (var(--brand-tint)), rounded-lg, padding. Show aiSummaryUpdatedAt as "Updated X ago" in caption text below.
       - If aiSummary is null: render graceful placeholder — "No AI summary yet. Click the refresh icon to generate one, or add notes to build context." in muted italic text. Do NOT auto-trigger synthesis (per locked decision: AI synthesis never auto-triggered on page load).

    3. **Last Q&A answer section (conditional):**
       - Only visible when lastAnswer is not null
       - Show the answer text in a bordered div
       - If lastAnswer.insufficient_context: show a muted warning "Limited context available — answer may be incomplete"
       - Source citations: if lastAnswer.sources.length > 0, render each as a small card below the answer:
         - Source label (bold), content snippet (truncated to 100 chars), date (relative)
         - Use subtle background + left border for visual distinction

    4. **Input area:**
       - `<textarea>` with 2 rows, auto-resize on content (but max 6 rows)
       - Placeholder: "Add a note or ask a question..."
       - Below textarea: hint text in caption style: "End with ? to ask AI, otherwise saves as a note"
       - Submit on Enter (without Shift). Shift+Enter for newline.
       - Submit button (Send icon) to the right of textarea, disabled when mode !== 'idle' or input is empty
       - While mode is 'asking': show a small spinner next to the input
       - While mode is 'saving_note': show "Saving..." text

    5. **Recent notes preview (optional enhancement):**
       - Skip this — the timeline tab already shows notes. Don't duplicate.

    **Styling:**
    - Panel has a subtle left border: `border-l border-[var(--subtle-border)]` on desktop (lg), no border on mobile
    - Background: slightly different from main content — use `bg-[var(--card-bg)]` with a very subtle shadow
    - Padding: spacing.card (24px)
    - Fixed width on desktop (parent controls 320px via flex), full width on mobile

    **Anti-patterns to avoid:**
    - Do NOT auto-trigger synthesis on mount (locked decision)
    - Do NOT call ask API when account has < 3 context entries (backend handles this, but show a nicer message if the answer comes back with insufficient_context: true)
    - Do NOT store Q&A history — lastAnswer is only the most recent one. Previous answers are lost on component unmount (stateless per design).
