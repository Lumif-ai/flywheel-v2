  <action>
    1. **Fix _append_event_atomic in skill_executor.py (line 3210):** Change the SQL from:
       ```sql
       SET events_log = events_log || CAST(:event AS jsonb)
       ```
       to:
       ```sql
       SET events_log = COALESCE(events_log, '[]'::jsonb) || CAST(:event AS jsonb)
       ```
       This ensures that if events_log is NULL (bug state), the append initializes it as an empty array first, preventing silent event loss. No other changes to this function.

    2. **Extend MCP polling timeout in server.py (line 41):** Change the polling intervals from:
       ```python
       intervals = [3, 5, 8] + [10] * 27  # ~5 min total
       ```
       to:
       ```python
       intervals = [3, 5, 8] + [10] * 57  # ~10 min total
       ```
       This doubles the polling window from ~5 min to ~10 min, accommodating flywheel runs with up to 20 meetings being processed + prepped. The execution caps (Task 2) keep typical runs under 5 minutes, but edge cases need headroom.
  </action>
