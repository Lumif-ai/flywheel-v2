**Module docstring:** Explain this is the Gmail READ service, separate from the send-only service in google_gmail.py. Note that this service uses provider="gmail-read" and must NEVER touch provider="gmail" Integration rows.

**Scopes (3 scopes for the gmail-read grant):**
```python
SCOPES = [
    "https://www.googleapis.com/auth/gmail.readonly",
    "https://www.googleapis.com/auth/gmail.modify",
    "https://www.googleapis.com/auth/gmail.send",
]
```
Include gmail.send so the draft approval flow (Phase 4) can use this same credential.

**TokenRevokedException class** — same pattern as google_gmail.py. Can import from google_gmail.py if preferred, but a local class is simpler and avoids coupling.

**OAuth flow functions (mirror google_gmail.py exactly):**
- `_create_oauth_flow() -> Flow` — uses `settings.google_gmail_read_redirect_uri` (NOT `settings.google_gmail_redirect_uri`). Uses same `settings.google_client_id` and `settings.google_client_secret`.
- `generate_gmail_read_auth_url(state: str) -> str` — calls `flow.authorization_url(access_type="offline", prompt="consent", state=state)`. CRITICAL: do NOT pass `include_granted_scopes="true"` — per research, this must NOT be used for the gmail-read grant to keep credentials isolated.
- `exchange_gmail_read_code(code: str) -> Credentials` — exchanges code, validates refresh_token is present.

**Credential functions (copy from google_gmail.py):**
- `serialize_credentials(creds: Credentials) -> bytes`
- `deserialize_credentials(encrypted: bytes) -> Credentials`
- `get_valid_credentials(integration) -> Credentials` — with RefreshError handling and TokenRevokedException

**Gmail API operations (all wrapped in asyncio.to_thread):**

- `list_message_headers(creds: Credentials, page_token: str | None = None, label_ids: list[str] | None = None, max_results: int = 100) -> dict`
  - Calls `service.users().messages().list(userId="me", labelIds=label_ids or ["INBOX"], maxResults=max_results)`
  - If page_token provided, adds `pageToken` kwarg
  - Returns raw API response dict (has `messages`, `nextPageToken`, `resultSizeEstimate`)

- `get_message_headers(creds: Credentials, message_id: str) -> dict`
  - Calls `service.users().messages().get(userId="me", id=message_id, format="metadata", metadataHeaders=["From", "To", "Subject", "Date"])`
  - Returns raw API response dict

- `get_message_body(creds: Credentials, message_id: str) -> str`
  - Calls `service.users().messages().get(userId="me", id=message_id, format="full")`
  - Calls `_extract_body(msg)` helper to extract text/plain or text/html from payload parts
  - Returns body text string

- `_extract_body(msg: dict) -> str` (private helper)
  - Walks `msg["payload"]["parts"]` recursively
  - Prefers text/plain, falls back to text/html
  - Base64url-decodes the `body.data` field
  - Returns empty string if no text part found

- `list_sent_messages(creds: Credentials, max_results: int = 200) -> dict`
  - Calls `service.users().messages().list(userId="me", labelIds=["SENT"], maxResults=max_results)`
  - Returns raw API response dict

- `get_history(creds: Credentials, start_history_id: str) -> dict`
  - Calls `service.users().history().list(userId="me", startHistoryId=start_history_id, historyTypes=["messageAdded"])`
  - Returns raw API response dict
  - NOTE: Caller (Phase 2 sync worker) handles HttpError 404 for stale historyId

- `get_profile(creds: Credentials) -> dict`
  - Calls `service.users().getProfile(userId="me")`
  - Returns dict with emailAddress and historyId (used for initial historyId capture)

- In _extract_body, if parsing fails, log: `logger.warning("failed to extract body from message_id=%s", msg.get("id"))` — NOT the message content
