Header:
```python
"""create email copilot tables with RLS

Revision ID: 020_email_models
Revises: 019_documents
Create Date: 2026-03-24
"""
revision = "020_email_models"
down_revision = "019_documents"
```

The upgrade() function must execute raw SQL (via `op.execute()`) for each of the four tables. For EACH table follow this exact sequence from the 010_context_graph_tables.py pattern:

1. **CREATE TABLE** with all columns, types, defaults, and constraints
2. **CREATE INDEX** statements for each index defined in the ORM model's `__table_args__`
3. **ALTER TABLE ... ENABLE ROW LEVEL SECURITY**
4. **ALTER TABLE ... FORCE ROW LEVEL SECURITY**
5. **GRANT SELECT, INSERT, UPDATE, DELETE ON {table} TO app_user**
6. **CREATE POLICY** — four separate policies per table (tenant_isolation_select, tenant_isolation_insert, tenant_isolation_update, tenant_isolation_delete) using `current_setting('app.tenant_id', true)::uuid`
   - SELECT: `USING (tenant_id = current_setting(...))`
   - INSERT: `WITH CHECK (tenant_id = current_setting(...))`
   - UPDATE: `USING (tenant_id = current_setting(...))` (no WITH CHECK needed — tenant_id should not change)
   - DELETE: `USING (tenant_id = current_setting(...))`
7. **Verification DO block** — assert RLS is enabled (check `pg_class.relrowsecurity`)
8. **Attach set_updated_at() trigger** for tables with `updated_at` column (email_drafts, email_voice_profiles). The trigger function already exists. Pattern: `CREATE TRIGGER set_updated_at BEFORE UPDATE ON {table} FOR EACH ROW EXECUTE FUNCTION set_updated_at()`

Column types in SQL:
- UUID PRIMARY KEY DEFAULT gen_random_uuid()
- UUID NOT NULL REFERENCES tenants(id) for tenant_id
- UUID NOT NULL REFERENCES users(id) for user_id
- UUID NOT NULL REFERENCES emails(id) for email_id
- UUID REFERENCES context_entities(id) for sender_entity_id (nullable)
- TEXT, TEXT NOT NULL as appropriate
- INTEGER, INTEGER NOT NULL DEFAULT 0
- BOOLEAN NOT NULL DEFAULT false
- JSONB NOT NULL DEFAULT '[]'::jsonb or '{}'::jsonb
- TIMESTAMPTZ NOT NULL DEFAULT now() or TIMESTAMPTZ (nullable)
- TEXT[] NOT NULL DEFAULT '{}'::text[] for labels array

The downgrade() function should DROP all four tables in reverse order (email_voice_profiles, email_drafts, email_scores, emails) — dropping tables automatically drops their RLS policies, indexes, and triggers.

CRITICAL: Do NOT log or reference email content anywhere in this migration. The `emails` table intentionally has no `body` column.
