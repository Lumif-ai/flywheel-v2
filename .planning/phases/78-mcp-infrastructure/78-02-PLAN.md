  <action>
Add centralized tool invocation logging to server.py using FastMCP's Middleware system.

1. Add imports at the top:
   - import logging
   - from fastmcp.server.middleware import Middleware

2. Set up a logger that writes to stderr (critical -- MCP uses stdout for protocol):
   ```python
   logger = logging.getLogger("flywheel_mcp")
   if not logger.handlers:
       import sys
       handler = logging.StreamHandler(sys.stderr)
       handler.setFormatter(logging.Formatter("%(asctime)s %(levelname)s %(message)s"))
       logger.addHandler(handler)
       logger.setLevel(logging.INFO)
   ```

3. Create InvocationLogger class (place before the mcp = FastMCP(...) line):
   ```python
   class InvocationLogger(Middleware):
       async def on_call_tool(self, context, call_next):
           tool_name = context.message.name
           args = context.message.arguments
           start = time.time()
           try:
               result = await call_next(context)
               duration = time.time() - start
               logger.info(
                   "tool_call tool=%s params=%s duration_ms=%d success=true",
                   tool_name, args, round(duration * 1000),
               )
               return result
           except Exception as exc:
               duration = time.time() - start
               logger.error(
                   "tool_call tool=%s params=%s duration_ms=%d success=false error=%s",
                   tool_name, args, round(duration * 1000), exc,
               )
               raise
   ```

4. Register the middleware right after creating the FastMCP instance:
   ```python
   mcp = FastMCP("Flywheel")
   mcp.add_middleware(InvocationLogger())
   ```

IMPORTANT per research Pitfall 2: Since MCP tools catch all exceptions and return error strings, the middleware except block will rarely fire. The middleware still correctly logs timing and invocation for every call. This is the intended design -- tools handle their own error reporting, middleware handles observability.

Note: `time` is already imported in server.py (line 5). Do not duplicate the import.

If FastMCP Middleware import fails (version too old), fall back to a decorator-based approach per STATE.md decision "MCP logging via Python decorator (not FastMCP middleware -- version-independent)":
   ```python
   import functools
   def log_tool_call(fn):
       @functools.wraps(fn)
       def wrapper(*args, **kwargs):
           start = time.time()
           try:
               result = fn(*args, **kwargs)
               duration = time.time() - start
               logger.info("tool_call tool=%s duration_ms=%d success=true", fn.__name__, round(duration * 1000))
               return result
           except Exception as exc:
               duration = time.time() - start
               logger.error("tool_call tool=%s duration_ms=%d success=false error=%s", fn.__name__, round(duration * 1000), exc)
               raise
       return wrapper
   ```
   Then apply @log_tool_call before each @mcp.tool() decorator.
