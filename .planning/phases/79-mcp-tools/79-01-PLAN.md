**MCP-01: flywheel_fetch_skills**
- No parameters
- Docstring verbatim from spec: "List all available Flywheel skills with descriptions, categories, and trigger phrases. Call this to discover what Flywheel can do for the user, or to find the right skill for a user's request. Returns skill names, descriptions, categories, trigger phrases, and context store contracts (what each skill reads/writes). Use trigger phrases to match natural language requests to skills."
- Implementation: `client.fetch_skills()`, get `items` list
- Empty case: return "No skills available. Check that skills are seeded and enabled."
- Format each skill as:
  ```
  **{name}** [{category}]
    {description}
    Triggers: {comma-separated triggers}
    Reads: {contract_reads} | Writes: {contract_writes}
  ```
  Where category = tags[0] or "uncategorized", triggers from `s.get("triggers", [])`, contract reads/writes from `s.get("contract_reads", [])` / `s.get("contract_writes", [])`.
- Header line: `f"Found {len(skills)} available skills:\n"`
- Join skill blocks with `"\n\n"`
- Two-level except: FlywheelAPIError then generic Exception

- Return the raw system_prompt text as-is (no formatting -- Claude Code needs the full prompt)
- Two-level except: FlywheelAPIError then generic Exception
