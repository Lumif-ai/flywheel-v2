**VoiceAnnotation.tsx** — Collapsible voice profile display:

Props: `{ snapshot: VoiceSnapshot | null | undefined }`

1. If snapshot is null/undefined, return null (graceful handling for legacy drafts)
2. Default state: collapsed (`useState(false)`)
3. Collapsed view: Show a subtle section header "Voice applied" with a ChevronRight/ChevronDown toggle icon (use lucide-react). Display 5 key fields inline as small badges/tags: tone, greeting_style, sign_off, avg_length (as word count), characteristic_phrases (first 2 phrases)
4. Expanded view: Show all 10 voice fields in a 2-column grid layout. Each field shows its label and value. Use the same label formatting as VoiceProfileSettings.tsx for consistency. Fields with null values show "Not set" in muted text.
5. Styling: Use Tailwind. Muted background (bg-gray-50 or similar), rounded corners, text-xs/text-sm sizing. The section should feel informational, not dominant — it's secondary to the draft body.
6. Use the collapsible pattern from ThreadDetail.tsx (ChevronDown/ChevronRight toggle with smooth transition).

**RegenerateDropdown.tsx** — Quick action dropdown with custom option:

Props: `{ draftId: string, hasUserEdits: boolean, isRegenerating: boolean, onRegenerate: (req: RegenerateRequest) => void }`

1. Main button: "Regenerate" with a RefreshCw icon (lucide-react) and a ChevronDown for the dropdown indicator
2. Dropdown menu with 5 items:
   - "Shorter" (ArrowDown icon or similar)
   - "Longer" (ArrowUp icon)
   - "More casual" (Smile icon or similar)
   - "More formal" (Briefcase icon or similar)
   - Divider
   - "Custom..." (Pencil icon) — opens an inline text input
3. When a quick action is clicked: call `onRegenerate({ action: 'shorter' })` etc.
4. When "Custom..." is clicked: show an inline text input below the dropdown (or expand inline within the DraftReview area) with a placeholder "e.g., be more empathetic, add a question about timeline..." and a "Regenerate" submit button. On submit: call `onRegenerate({ custom_instructions: inputValue })`
5. If `hasUserEdits` is true and a regeneration is triggered: show a confirmation dialog first ("You have edits that will be replaced. Continue?") before calling onRegenerate
6. While `isRegenerating` is true: show a loading spinner on the button, disable the dropdown
7. Implement the dropdown using a simple div with click-outside-to-close pattern (same approach used elsewhere in the codebase), or use an existing dropdown primitive if one exists (check for Radix DropdownMenu in package.json)
