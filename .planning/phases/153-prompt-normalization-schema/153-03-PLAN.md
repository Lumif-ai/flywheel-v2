<task type="auto">
  <name>Task 1: Create validate_skill_tools.py CI validation script</name>
  <files>scripts/validate_skill_tools.py</files>
  <action>
Create `scripts/validate_skill_tools.py` -- a standalone CLI script that:

1. **Scans all skill directories** under `skills/`, skipping `_archived` and library skills (`_shared`, `gtm-shared`, `broker` -- those with `enabled: false`).

2. **For each active skill**, reads the SKILL.md, extracts `system_prompt` (everything after the closing `---` of frontmatter).

3. **Applies `normalize_for_mcp()`** to the extracted prompt (import from `backend/src/flywheel/services/prompt_normalizer.py` -- add `backend/src` to sys.path).

4. **Checks the NORMALIZED prompt** against a forbidden vocabulary list using word-boundary regex:

```python
FORBIDDEN_TOOLS = [
    # Server-side tool names (should be MCP equivalents after normalization)
    "context_read", "read_context", "context_write", "append_entry", "context_query",
    # Native capability server names (should be WebSearch/WebFetch/etc)
    "web_search", "web_fetch", "file_write", "file_read", "read_file", "write_file", "python_execute",
    # Deprecated lead tools (should be pipeline equivalents)
    "flywheel_list_leads", "flywheel_upsert_lead", "flywheel_add_lead_contact",
    "flywheel_draft_lead_message", "flywheel_send_lead_message",
    "flywheel_graduate_lead", "flywheel_fetch_account",
    # MCP namespace prefix (should be stripped)
    "mcp__flywheel__",
]
```

5. **Reports results:**
   - For each violation: `FAIL: {skill_name} -- found forbidden tool '{tool_name}' (line {N})`
   - Summary: `{N} skills checked, {M} violations found`
   - Exit 0 if zero violations, exit 1 if any violations

6. **CLI flags:**
   - `--verbose`: Show each skill being checked
   - `--skill NAME`: Check a single skill only (for quick iteration)

**Key design decisions:**
- The script imports `normalize_for_mcp` from the backend module (add `sys.path.insert` for `backend/src`). This ensures the validator uses the SAME normalization logic as the API.
- Forbidden check runs AFTER normalization. This means the script validates what the API would actually serve, not just the raw prompt.
- Use the same `parse_frontmatter` pattern from `scripts/validate_skills.py` for YAML parsing.
- Word-boundary matching for all checks except `mcp__flywheel__` which is a prefix check.

Make the script executable with `#!/usr/bin/env python3` and proper `if __name__ == "__main__"` block.
