   This creates `frontend/src/components/ui/sonner.tsx` with a themed Toaster wrapper. If the shadcn command fails or is not available, install manually: `npm install sonner` and create a simple Toaster re-export.

2. **layout.tsx** — Mount the Toaster in the AppShell:
   - Import `{ Toaster }` from `@/components/ui/sonner` (or from `sonner` directly if shadcn wrapper not created)
   - Add `<Toaster position="top-right" />` as a sibling of `<CommandPalette />` inside the `<TooltipProvider>` wrapper — this ensures toasts appear on ALL pages, not just the Email page
   - Do NOT modify any other layout logic

3. **CriticalEmailAlert.tsx** — Renderless component that fires Sonner toasts for priority-5 threads:
   - Props: threads (Thread[]) — the full thread list from useEmailThreads
   - Import `{ toast }` from `sonner`, useEmailStore for alertDismissedIds and dismissAlert, and `useNavigate` from react-router
   - In a useEffect dependent on [threads, alertDismissedIds, dismissAlert]:
     - Filter threads to those with max_priority === 5 AND thread_id NOT in alertDismissedIds
     - For each critical thread, call:
       ```
       toast.warning(`Critical: ${thread.subject ?? thread.sender_email}`, {
         id: thread.thread_id,       // Sonner deduplicates by id — prevents re-fire
         duration: Infinity,         // Persist until user dismisses
         onDismiss: () => dismissAlert(thread.thread_id),
         action: {
           label: 'View',
           onClick: () => navigate(`/email?thread=${thread.thread_id}`),
         },
       })
       ```
     - Immediately call dismissAlert(thread.thread_id) after toast() — this marks it as "alerted" so the NEXT useEffect pass (when threads refetches every 30s) skips it. Sonner's id-based deduplication handles the current render; alertDismissedIds handles subsequent refetches.
   - Return null (renderless component)

4. **Mount CriticalEmailAlert globally** — this component needs to run even when the user is NOT on /email:
   - In layout.tsx's AppShell (the non-standalone branch), add:
     - Import useEmailThreads from the email hooks
     - Call `const { data } = useEmailThreads()` (this is a lightweight GET that React Query caches at 30s stale time — minimal overhead)
     - Render `{data?.threads && <CriticalEmailAlert threads={data.threads} />}` alongside the Toaster
   - This ensures priority-5 alerts fire on ANY page. The query is shared with the EmailPage via React Query's cache — no duplicate network requests.
  </action>

  <done>Sonner Toaster mounted at app root. Priority-5 emails trigger persistent toast alerts on any page. Toasts are deduplicated by thread_id. Dismissed alerts are tracked in Zustand and never re-fire.</done>
