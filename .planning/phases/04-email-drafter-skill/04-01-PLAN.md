**Constants and imports:**
- `_SONNET_MODEL = "claude-sonnet-4-6"` (NOT Haiku -- drafting is quality-sensitive per research)
- Import: `anthropic`, `json`, `re`, `logging`, `datetime`, `UUID`, `AsyncSession`
- Import from models: `Email`, `EmailScore`, `EmailDraft`, `EmailVoiceProfile`, `ContextEntry`, `ContextEntity`
- Import from config: `settings`
- Import from gmail_read: `get_message_body`, `get_valid_credentials`
- Import from `sqlalchemy.dialects.postgresql import insert as pg_insert`
- Import `HttpError` from `googleapiclient.errors` for 401/403 handling

**DEFAULT_VOICE_STUB** (used when no EmailVoiceProfile exists):
```python
DEFAULT_VOICE_STUB = {
    "tone": "professional and direct",
    "avg_length": 80,
    "sign_off": "Best,",
    "phrases": [],
}
```

**DRAFT_SYSTEM_PROMPT** -- voice profile fields are injected via `.format()`:
- Tone, avg_length, sign_off, phrases injected as structured constraints
- Context block placeholder for assembled context entries
- Reply constraints: address the ask directly, no subject line, no "I hope this email finds you well", no bullet points unless incoming used them, end with sign-off
- Output instruction: return only the reply body text, no JSON wrapping, no metadata

**Internal helpers (private functions):**

1. `_load_voice_profile(db, tenant_id, user_id) -> dict` -- Query EmailVoiceProfile for the tenant+user. If found, return dict with tone, avg_length, sign_off, phrases (cap phrases at 5 items). If not found, log a warning and return DEFAULT_VOICE_STUB.

2. `_assemble_draft_context(db, tenant_id, context_refs) -> str` -- Takes the `context_refs` list from `EmailScore.context_refs` (already populated by scorer). Extract entry UUIDs (type="entry") and entity UUIDs (type="entity"). Load up to 5 ContextEntry rows by UUID and up to 3 ContextEntity rows by UUID. Format as a text block:
   ```
   [Meeting note from {file_name}, {date}]: "{content_snippet[:300]}"
   [Entity: {name} ({entity_type}, {mention_count} mentions)]: {detail or aliases summary}
   ```
   If no refs or no matches found, return "No relevant context available."

3. `_fetch_body_with_fallback(integration, email, creds) -> tuple[str, str | None]` -- Calls `get_message_body(creds, email.gmail_message_id)`. Wrap in try/except for `HttpError`. If `exc.resp.status in (401, 403)`: fall back to `email.snippet or ""`, return `(snippet, "body_fetch_failed:{status}")`. For other HttpErrors, re-raise. Also check: if `len(body_text.strip()) < 20 and len((email.snippet or '').strip()) < 10`, return `("", "body_too_short")` -- skip drafting for empty bodies. Return `(body_text, None)` on success.

4. `_build_draft_prompt(email, body_text, voice_profile, context_block) -> tuple[str, str]` -- Build system prompt by formatting DRAFT_SYSTEM_PROMPT with voice profile fields and context block. Build user message with: From, Subject, email body text. Return (system_prompt, user_message).

5. `_upsert_email_draft(db, tenant_id, email_id, draft_body, context_used, fetch_error) -> None` -- Insert an EmailDraft row. Use simple INSERT (NOT on_conflict -- no unique constraint on email_id exists in migration). The caller already checks for existing draft via LEFT JOIN IS NULL guard. Set `visible_after = now() + timedelta(days=settings.draft_visibility_delay_days)` if delay > 0, else NULL (immediately visible). Store `context_used` as JSONB list of context refs. If `fetch_error` is not None, include `{"fetch_error": fetch_error}` in the context_used list. Status = "pending". Does NOT commit -- caller-commits pattern.

**Main entry point:**

`async def draft_email(db, tenant_id, email, score, integration, api_key=None) -> dict | None`

Pipeline:
1. Load voice profile via `_load_voice_profile(db, tenant_id, email.user_id)`
2. Get valid credentials from integration via `get_valid_credentials(integration)`
3. Fetch body with fallback via `_fetch_body_with_fallback(integration, email, creds)`
4. If body is empty AND fetch_error is "body_too_short", return None (skip draft, not an error)
5. Assemble context from `score.context_refs` via `_assemble_draft_context(db, tenant_id, score.context_refs)`
6. Build prompt via `_build_draft_prompt`
7. Call Sonnet via AsyncAnthropic with `max_tokens=1000` (cap draft length)
8. Extract response text -- NO JSON parsing needed (draft is raw text, not JSON)
9. Truncate if >2000 chars, log warning
10. Upsert EmailDraft via `_upsert_email_draft`
11. Log success: email.id, tenant_id (no PII)
12. Return dict with `{"draft_body": draft_body, "context_used": context_refs_list}`

**Error boundary (wrap entire function):**
- Outer try/except catches ALL exceptions
- Log: `draft_email failed for email_id={email.id} tenant_id={tenant_id}: {type}: {msg}` (no PII -- no subject/sender/body in log)
- Return None on any failure
- Non-fatal: caller always continues
