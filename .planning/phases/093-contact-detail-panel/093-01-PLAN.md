  <action>
    **api.ts** -- Add 4 new functions at the end of the file (after existing exports):

    1. `fetchContactActivities(entryId: string, contactId: string)` -- calls `api.get('/pipeline/${entryId}/activities', { params: { contact_id: contactId, limit: 100 } })`, returns `Promise<{ items: PipelineActivity[]; total: number }>`.

    2. `patchContact(entryId: string, contactId: string, data: Record<string, unknown>)` -- calls `api.patch('/pipeline/${entryId}/contacts/${contactId}', data)`, returns `Promise<PipelineContact>`.

    3. `patchActivity(entryId: string, activityId: string, data: Record<string, unknown>)` -- calls `api.patch('/pipeline/${entryId}/activities/${activityId}', data)`, returns `Promise<PipelineActivity>`.

    4. `createActivity(entryId: string, data: { type: string; channel?: string; status?: string; subject?: string; body_preview?: string; contact_id?: string; metadata_?: Record<string, unknown> })` -- calls `api.post('/pipeline/${entryId}/activities', data)`, returns `Promise<PipelineActivity>`.

    **types/pipeline.ts** -- No changes needed. `PipelineActivity`, `PipelineContact`, and `ContactListItem` already have the required shapes.

    **hooks/useContactActivities.ts** -- Create a new query hook:
    - `useContactActivities(entryId: string | null, contactId: string | null)` 
    - Uses `useQuery` with queryKey `['contact-activities', contactId]`
    - `enabled: !!entryId && !!contactId`
    - Calls `fetchContactActivities(entryId!, contactId!)`
    - Returns the query result

    **hooks/useContactMutation.ts** -- Create mutation hook following `usePipelineMutation.ts` pattern:
    - `useContactMutation()`
    - `mutationFn` accepts `{ entryId: string; contactId: string; data: Record<string, unknown> }`
    - Calls `patchContact(entryId, contactId, data)`
    - `onError`: `toast.error('Failed to update contact')`
    - `onSettled`: invalidate `['contacts']` and `['contact-activities']` query keys

    **hooks/useActivityMutation.ts** -- Create mutation hook:
    - `useActivityMutation()`
    - `mutationFn` accepts `{ entryId: string; activityId: string; data: Record<string, unknown> }`
    - Calls `patchActivity(entryId, activityId, data)`
    - `onError`: `toast.error('Failed to update activity')`
    - `onSettled`: invalidate `['contacts']` and `['contact-activities']` query keys
    - `onSuccess`: `toast.success('Activity updated')` (only for status changes -- check if `data.status` exists)

    **hooks/useCreateActivity.ts** -- Create mutation hook:
    - `useCreateActivity()`
    - `mutationFn` accepts `{ entryId: string; data: { type: string; channel?: string; status?: string; subject?: string; body_preview?: string; contact_id?: string; metadata_?: Record<string, unknown> } }`
    - Calls `createActivity(entryId, data)`
    - `onError`: `toast.error('Failed to create step')`
    - `onSuccess`: `toast.success('New step created')`
    - `onSettled`: invalidate `['contacts']` and `['contact-activities']` query keys
  </action>
