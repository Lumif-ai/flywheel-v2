    }

    - Fixed position: `position: fixed; top: 0; right: 0; width: 480px; height: 100vh; z-index: 40` (480px, wider than company panel's 420px per research recommendation for textarea comfort)
    - Background white, border-left `1px solid #E5E7EB`, `box-shadow: -4px 0 12px rgba(0,0,0,0.05)`

    - Escape key listener via `useEffect` to call `onClose`
    - Overflow-y: auto on the content area below the header

    **Section 1 -- Contact Header (PANEL-01):**
    - Fixed header area (not scrollable) with close button (X icon) at top-right
    - Contact name as the main heading (18px, bold)
    - Below name: title + company_name in gray
    - Editable fields in a grid: email, phone, linkedin_url
    - Each field: display as text by default, subtle pencil icon on hover, click to switch to input
    - On blur or Enter: call `contactMutation.mutate({ entryId: contact.pipeline_entry_id, contactId: contact.id, data: { [field]: value } })`
    - Use `useContactMutation()` hook from Plan 01
    - Show `toast.success('Saved')` is handled by the hook -- no need to add here

    **Section 2 -- Outreach Sequence Timeline (PANEL-02, PANEL-03, PANEL-04, PANEL-05):**
    - Section header: "OUTREACH SEQUENCE" in 11px uppercase gray (matches PipelineSidePanel section header pattern)
    - Fetch activities via `useContactActivities(contact.pipeline_entry_id, contact.id)`
    - Sort activities by `metadata.step_number` ascending (fallback to `occurred_at` if no step_number)
    - For each activity, render a timeline step:

      **Timeline visual:**
      - Vertical line on the left (absolute positioned, 1px, #E5E7EB)
      - Dot marker per step (10px circle, colored by status: drafted=#9CA3AF, approved=#D97706, sent=#2563EB, replied=#059669, bounced=#DC2626, skipped=#D1D5DB)
      - Content to the right of the line

      **Step header row:**
      - "Step {metadata.step_number}" label (or "Step ?" if no step_number)
      - Channel icon from lucide-react: Mail for email, Linkedin icon (use `MessageSquare` as substitute) for linkedin, Phone for phone
      - Subject line (truncated if long)
      - Status pill: colored badge with status text. Use colors: drafted=#F3F4F6/#6B7280, approved=#FEF3C7/#D97706, sent=#DBEAFE/#2563EB, replied=#D1FAE5/#059669, bounced=#FEE2E2/#DC2626, skipped=#F3F4F6/#9CA3AF

      **Step body (PANEL-03, PANEL-04):**
      - Editable textarea showing `activity.body_preview` (the message body)
      - Textarea: min-height 80px, resize vertical, border `1px solid #E5E7EB`, rounded 8px, padding 12px, font-size 13px
      - Save on blur (NOT on every keystroke -- use local state, save on blur)
      - On blur: if value changed, call `activityMutation.mutate({ entryId: contact.pipeline_entry_id, activityId: activity.id, data: { body_preview: newValue } })`
      - If activity has both email channel content and subject, show subject as editable input above the textarea
      - Use `useActivityMutation()` hook from Plan 01

      **Step action buttons (PANEL-05):**
      - Row of 3 buttons below the textarea:
        - "Approve" (CheckCircle icon) -- sets status to "approved". Only show when status is "drafted". Use green accent.
        - "Skip" (SkipForward icon) -- sets status to "skipped". Show when status is "drafted" or "approved". Use gray.
        - "Mark Replied" (MessageSquare icon) -- sets status to "replied". Show when status is "sent". Use green accent.
      - On click: `activityMutation.mutate({ entryId, activityId, data: { status: newStatus } })`
      - Disable button while mutation is pending
      - Visual feedback: button shows loading state (opacity 0.5) during mutation

    **Section 3 -- Generate Next Step (PANEL-06):**
    - Below the timeline, a "Generate Next Step" button (Plus icon, full-width, dashed border, gray text)
    - On click: calculate `nextStepNumber = max(step_numbers) + 1` (or 1 if no steps)
    - Call `createActivityMutation.mutate({ entryId: contact.pipeline_entry_id, data: { type: 'message', status: 'drafted', contact_id: contact.id, metadata_: { step_number: nextStepNumber }, subject: '', body_preview: '' } })`
    - Use `useCreateActivity()` hook from Plan 01
    - After creation, the query invalidation (handled by hook) will refetch and show the new step in the timeline

    **State management:**
    - Reset all local edit state when `contact.id` changes (useEffect with contactId dependency)
    - Use `useContactActivities` queryKey `['contact-activities', contactId]` so switching contacts fetches fresh data
    - Loading state: show skeleton/spinner while activities are loading
    - Empty state: if no activities, show "No outreach steps yet" message above the Generate button

    **Styling notes:**
    - Use inline styles (matching codebase convention from PipelineSidePanel)
    - Font: inherit from app (Geist Variable)
    - Colors: #121212 headings, #6B7280 body text, #9CA3AF labels
    - No Tailwind utility classes for layout (match PipelineSidePanel inline style pattern), but Tailwind for flex/gap/padding helpers is fine
