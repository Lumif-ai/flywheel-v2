<objective>
Build the meeting intelligence pipeline: a 7-stage engine that fetches transcripts from Granola, uploads to Supabase Storage, classifies meeting type via 3-layer detection (contact match -> internal check -> Haiku LLM), extracts 9 insight types via Sonnet, and writes structured intelligence to ContextEntry rows. Includes the POST /meetings/{id}/process API endpoint and skill_executor dispatch wiring.

Purpose: This is the intelligence brain that transforms raw meeting data (from Phase 60) into structured CRM intelligence. Without this, meetings are just metadata rows with no extracted value.

Output: Working meeting processing pipeline triggered via POST endpoint, dispatched through job_queue, producing ContextEntry rows and meeting.summary JSONB.
</objective>

  <name>Task 1: Meeting processor web engine module with classify, extract, upload, and context write helpers</name>
  <files>backend/src/flywheel/engines/meeting_processor_web.py</files>
  <action>
Create `backend/src/flywheel/engines/meeting_processor_web.py` with these functions:

1. **classify_meeting()** — 3-layer classification returning one of 8 types (discovery, expert, prospect, advisor, investor-pitch, internal, customer-feedback, team-meeting):
   - Layer 1: Query `AccountContact` for attendee emails. If found, derive type from parent account's `relationship_type` array (e.g., advisor account -> "advisor" type). Use `factory()` session with RLS context set.
   - Layer 2: If all attendee emails share the tenant domain -> classify as internal (<=3 attendees) or team-meeting (>3). Query `Tenant.domain` from DB. If `tenant.domain` is NULL, skip Layer 2 entirely (log debug warning).
   - Layer 3: Haiku LLM call with title + attendees + first 500 chars of transcript. Model string: `"claude-haiku-4-5-20251001"`. Use `anthropic.Anthropic()` sync client wrapped in `asyncio.get_event_loop().run_in_executor(None, lambda: ...)` to avoid blocking event loop. Prompt returns ONLY the classification code.
   - Accept `api_key: str | None` param — pass to `anthropic.Anthropic(api_key=api_key)` if provided.

2. **extract_intelligence()** — Sonnet extraction of 9 insight types into structured JSON dict:
   - Model string: `"claude-sonnet-4-20250514"`. Sync client in run_in_executor.
   - System prompt instructs extraction of: tldr, key_decisions, action_items, pain_points, buying_signals, competitor_mentions, budget_signals, champion_indicators, next_steps.
   - User prompt includes: full transcript, provider's ai_summary (if available), meeting_type, existing account context (if any).
   - Parse JSON response (strip ```json fences if present).
   - Return dict with all 9 keys (empty lists for missing keys).

3. **upload_transcript()** — Upload transcript text to Supabase Storage:
   - Path: `transcripts/{tenant_id}/{meeting_id}.txt` in the `uploads` bucket.
   - URL: `{SUPABASE_URL}/storage/v1/object/uploads/transcripts/{tenant_id}/{meeting_id}.txt`
   - Use httpx async client with `Authorization: Bearer {SUPABASE_SERVICE_ROLE_KEY}`, `Content-Type: text/plain`.
   - Return the storage path string.

4. **write_context_entries()** — Write extracted intelligence to ContextEntry rows:
   - Accept: factory, tenant_id, user_id, account_id (may be None), meeting_date, meeting_slug, extracted dict.
   - Map extracted keys to file_names (no .md extension): pain-points, buying-signals, competitor-intel, action-items, key-decisions, budget-signals, champion-indicators.
   - For each non-empty extracted key, create a ContextEntry ORM object directly:
     - `source="ctx-meeting-processor"`, `detail=meeting_slug`, `confidence="medium"`
     - `account_id` set directly on ORM object
     - `date` = meeting_date as datetime.date
   - Set RLS context (both app.tenant_id and app.user_id) in session before writes.
   - Dedup: before inserting, check if ContextEntry with same (file_name, source, detail, tenant_id) exists. Skip if so.
   - meeting_slug format: `meeting-{YYYY-MM-DD}-{title_slug}` where title_slug = first 20 chars of title, lowercased, non-alphanumeric replaced with hyphens.

Constants at module level:
- `_HAIKU_MODEL = "claude-haiku-4-5-20251001"`
- `_SONNET_MODEL = "claude-sonnet-4-20250514"`
- `EXTRACTION_SYSTEM_PROMPT` and `EXTRACTION_USER_PROMPT` as formatted string templates.
- `CONTEXT_FILE_MAP` dict mapping extracted keys to ContextEntry file_names.

Do NOT import from the existing CLI `engines/meeting_processor.py` — it uses sync storage_backend. The two implementations coexist permanently.
