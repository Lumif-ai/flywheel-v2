**1. Add apply_post_classification_rules() to meeting_processor_web.py:**

Place this function in `meeting_processor_web.py` (co-located with other processing helpers, avoids circular imports with api/meetings.py):

```python
def apply_post_classification_rules(
    meeting_type: str,
    external_id: str,
    attendees: list[dict],
    processing_rules: dict,
    tenant_domain: str | None = None,
) -> str:
```

Logic (4 rule types per MPP-05 spec):

- **Rule 1 — skip_meetings (manually skipped):** If `external_id in rules.get("manually_skipped", [])`, return "skipped". This covers specific meetings the user has excluded.

- **Rule 2 — skip_internal (default: ON per spec):** If `rules.get("skip_internal", True)` is truthy, check attendees for external participants. An attendee is external if `a.get("is_external", False)` is True. If NO attendees are external, return "skipped". This prevents wasting LLM credits on internal-only meetings.

- **Rule 3 — skip_domains:** Get `skip_domains = set(rules.get("skip_domains", []))`. If skip_domains is non-empty, collect all attendee email domains: `domains = {a.get("email", "").split("@")[-1] for a in attendees if a.get("email")}`. Discard empty strings from the set. If `domains` is non-empty and `domains.issubset(skip_domains | ({tenant_domain} if tenant_domain else set()))`, return "skipped". This skips meetings where all attendees are from domains the user wants to ignore (e.g., family.com, personal.net).

- **Rule 4 — skip_types (skip by meeting type):** If `meeting_type in rules.get("skip_meeting_types", [])`, return "skipped". Note the spec key is `skip_meeting_types` (not `skip_types`).

- Default: return "pending" (continue processing).

This is a pure function (no DB, no async needed). It reads from the `processing_rules` dict which comes from `Integration.settings['processing_rules']`.

**2. Add POST /meetings/process-pending batch endpoint in meetings.py:**

```python
    user: TokenPayload = Depends(require_tenant),

) -> dict:
```

Logic:
- Query all meetings WHERE `tenant_id = user.tenant_id AND processing_status = 'pending' AND deleted_at IS NULL`.

- Commit all at once (batch).
- Return `{"queued": count, "run_ids": [str(run.id) for run in runs]}`.
- If no pending meetings: return `{"queued": 0, "run_ids": []}` (200 OK, not an error).

Import `SkillRun` from `flywheel.db.models`.
