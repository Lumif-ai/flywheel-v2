  <name>Task 1: Account auto-linking helpers in meeting_processor_web.py</name>
  <files>backend/src/flywheel/engines/meeting_processor_web.py</files>
  <action>
Add three async functions to `meeting_processor_web.py`:

1. **auto_link_meeting_to_account(factory, tenant_id, attendees) -> UUID | None**
   - Extract external email domains from attendees list (dicts with email, name, is_external keys).
   - Strip common mail subdomains: mail., email., smtp., mx., www. prefixes.
   - Exclude common free email providers (gmail.com, yahoo.com, hotmail.com, outlook.com, icloud.com, protonmail.com, aol.com) — these should NOT create accounts.
   - Query `Account.domain.in_(external_domains)` with `Account.tenant_id == tenant_id`.
   - Set RLS context (app.tenant_id) in session before query.
   - If single match: return account.id.
   - If multiple matches: return the one with the most AccountContacts (or first if tied).
   - If no match: for each unique external domain (excluding free email providers), call `auto_create_prospect()`. Return the first created account's id.

2. **auto_create_prospect(factory, tenant_id, user_id, domain, attendees, title) -> UUID**
   - Derive company_name from domain: `domain.split(".")[0].title()`.
   - Compute normalized_name: `re.sub(r"[^a-z0-9]", "", company_name.lower())`.
   - Dedup check: query Account with `tenant_id + normalized_name`. If exists, return existing.id.
   - Create Account with ALL required NOT NULL fields:
     - `name=company_name`, `normalized_name=normalized`, `domain=domain`
     - `source="meeting-auto-discovery"`
     - `status="prospect"` (legacy column — still NOT NULL in current schema)
     - `relationship_type=["prospect"]`
     - `relationship_status="new"`, `pipeline_stage="identified"`
     - `last_interaction_at=datetime.now(timezone.utc)`
   - Set RLS context (app.tenant_id) before insert.
   - Flush to get account.id.
   - Call `upsert_account_contacts()` for attendees matching this domain.
   - Commit and return account.id.

3. **upsert_account_contacts(session, tenant_id, account_id, attendees, domain) -> int**
   - Filter attendees to those with `is_external=True` and email ending in `@{domain}`.
   - For each: check if AccountContact with `(tenant_id, account_id, email)` already exists.
   - If not: create `AccountContact(tenant_id, account_id, name, email, source="meeting-auto-discovery")`.
   - Return count of newly created contacts.

Key constraints from research:
- Use `content.attendees` from get_meeting_content() (passed in by caller), NOT meeting.attendees from DB row.
- Account has UniqueConstraint("tenant_id", "normalized_name") — always check before creating.
- AccountContact needs tenant_id, account_id, name, email at minimum.
- All session operations must have RLS context set (app.tenant_id via set_config).
