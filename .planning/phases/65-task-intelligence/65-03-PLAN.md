TaskResponse:
- All Task fields serialized (id, tenant_id, user_id, meeting_id, account_id, title, description, source, task_type, commitment_direction, suggested_skill, skill_context, trust_level, status, priority, due_date, completed_at, metadata, created_at, updated_at)

TasksListResponse:
- tasks: list[TaskResponse]
- total: int

TaskSummaryResponse:
- detected: int
- in_review: int
- confirmed: int
- in_progress: int
- done: int
- blocked: int
- dismissed: int
- overdue: int

**Status transition map (module-level constant):**
```python
VALID_TRANSITIONS: dict[str, set[str]] = {
    "detected":    {"in_review", "confirmed", "dismissed"},
    "in_review":   {"confirmed", "dismissed"},
    "confirmed":   {"in_progress", "dismissed"},
    "in_progress": {"done", "blocked", "dismissed"},
    "blocked":     {"in_progress", "dismissed"},
    "done":        set(),
    "dismissed":   {"detected"},
}
```

**7 Endpoints:**

1. `GET /` — List tasks with optional filters: status (str), commitment_direction (str), priority (str), meeting_id (UUID), account_id (UUID). Paginate with offset/limit (default 0/50). Order by created_at desc. Return TasksListResponse with total count.

2. `GET /summary` — Return TaskSummaryResponse with count per status plus overdue count (due_date < now AND status not in done/dismissed). NOTE: Define this route BEFORE /{task_id} to avoid path parameter conflict.

3. `GET /{task_id}` — Return single TaskResponse. RLS handles scoping — if not found, return 404.

4. `POST /` — Create manual task. Set source="manual", status="detected". Return 201 with TaskResponse.

5. `PATCH /{task_id}` — Update allowed fields (title, description, priority, due_date, suggested_skill, trust_level). Set updated_at=now(). Return TaskResponse. 404 if not found.

6. `PATCH /{task_id}/status` — Accept body with `status: str`. Validate transition: if current_status -> new_status not in VALID_TRANSITIONS, return 422 with error message listing valid targets. If transitioning to "done", set completed_at=now(). If transitioning FROM "done" (reopen via dismissed->detected), clear completed_at. Set updated_at=now(). Return TaskResponse.

7. `DELETE /{task_id}` — Soft delete: set status="dismissed", updated_at=now(). Return 204. 404 if not found.

**Dependencies:** Use `get_tenant_db` and `require_tenant` from deps.py (same as accounts.py). User scoping is enforced by RLS — no need for manual user_id WHERE clauses in queries.
