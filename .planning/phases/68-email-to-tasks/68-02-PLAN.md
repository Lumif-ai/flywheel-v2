Create `backend/src/flywheel/engines/channel_task_extractor.py` with the following components:

**1. CandidateTask TypedDict** (channel-agnostic shape):
```python
class CandidateTask(TypedDict):
    title: str
    description: str
    source: str              # "email", "slack", etc.
    source_id: UUID          # email.id, slack_message.id, etc.
    account_id: UUID | None
    task_type: str

    suggested_skill: str | None
    skill_context: dict | None

    priority: str

```

**2. `extract_channel_tasks()` -- Pure data function (NO SSE emission):**
- Accepts `factory`, `tenant_id`, `user_id`, `extractors` list (default: `[extract_email_tasks]`)
- Opens a session, sets RLS context (same pattern as `write_task_rows` in meeting_processor_web.py -- `set_config('app.tenant_id', ...)` and `set_config('app.user_id', ...)`)
- Determines `last_ritual_at` by querying: `SELECT MAX(created_at) FROM skill_runs WHERE skill_name='flywheel' AND status='complete' AND tenant_id=:tid AND user_id=:uid`
- Calls each extractor, collecting candidates and per-channel summaries
- For each candidate: runs dedup check, creates Task ORM object (see field mapping below), flushes batch
- Builds `tasks_detail` list (see section 6 below) alongside task creation
- Commits once at end
- Returns aggregated summary dict: `{"total_scored": N, "tasks_created": M, "duplicates_found": D, "skipped_existing": S, "is_first_run": bool, "channels": {"email": {...}}, "tasks_detail": [...]}`
- **This function does NOT call `_append_event_atomic` or emit SSE events.** It is a pure data function. The ritual wrapper in Task 2 handles all SSE emission.

**3. `extract_email_tasks()` -- Email channel extractor:**
- Signature: `async def extract_email_tasks(session, tenant_id, user_id, last_ritual_at) -> tuple[list[CandidateTask], dict]`
- Query qualifying emails (use SQLAlchemy ORM, not raw SQL):
  - JOIN `Email` with `EmailScore` on `email_id`
  - WHERE `tenant_id = :tid AND user_id = :uid`  (on Email table -- it has both)
  - WHERE `EmailScore.category.in_(['action_required', 'meeting_followup'])`
  - WHERE `EmailScore.priority >= 4`
  - WHERE `EmailScore.scored_at > last_ritual_at` (or `> now() - 7 days` if `last_ritual_at` is None)
  - WHERE `NOT EXISTS (SELECT 1 FROM tasks t WHERE t.email_id = e.id)` -- idempotency via FK column
  - ORDER BY `EmailScore.priority DESC, Email.received_at DESC`
  - LIMIT: 15 if first run (`last_ritual_at is None`), else 100
- **Important:** EmailScore has NO user_id column. Filter user_id via the Email table join only.
- For each qualifying email+score row, build a `CandidateTask`:
  - `title`: Strip `Re:`, `Fwd:`, `Fw:` prefixes (case-insensitive regex: `r'^(Re|Fwd|Fw)\s*:\s*'`). If subject is None/empty, use `"(No subject) -- from {sender_name}"`
  - `description`: `f"From: {sender_name} ({sender_email})\n{snippet}\n\nScorer reasoning: {reasoning}"`
  - `source`: `"email"`
  - `source_id`: `email.id`
  - `task_type`: `"followup"` (both `action_required` and `meeting_followup` map to followup)
  - `commitment_direction`: `"yours"` if `category=='action_required'`, `"mutual"` if `category=='meeting_followup'`
  - `priority`: `"high"` if `score.priority == 5`, else `"medium"` (priority 4)
  - `trust_level`: `"review"` by default. Override to `"confirm"` if `suggested_skill` contains "email" (case-insensitive)
  - `suggested_skill`: `"email-drafter"` if `suggested_action == "draft_reply"`, else None
  - `skill_context`: `{"email_id": str(email.id), "gmail_thread_id": email.gmail_thread_id}` if suggested_skill is set, else None
  - `email_id`: `email.id`
  - `account_id`: resolved via `_resolve_entity_to_account()` (see below)
  - `metadata`: `{"email_score_id": str(score.id), "gmail_thread_id": email.gmail_thread_id}`
- Return `(candidates, {"total_scored": len(candidates), "is_first_run": last_ritual_at is None})`

**4. `_resolve_entity_to_account()` -- Entity-to-account helper:**
- First, check grep results from the search step above. If an existing helper was found, import and use it instead of implementing this.
- If no existing helper: Takes `session`, `tenant_id`, `sender_entity_id` (UUID or None)
- If `sender_entity_id` is None, return None
- Query `ContextEntity` by id. If not found or deleted, return None
- If `entity_type == "company"`: look up Account where `Account.name == entity.name AND Account.tenant_id == tenant_id` (simple name match). Return account_id if found.
- If `entity_type == "person"`: look up `AccountContact` where `AccountContact.name == entity.name AND AccountContact.tenant_id == tenant_id`. If found, return `contact.account_id`.
- If no match, return None
- This is a best-effort resolution. Log at DEBUG level, never raise.

**5. `_normalize_title()` and `_find_duplicate()` -- Dedup guard:**

`_normalize_title(title: str) -> set[str]`:
- Strip `Re:`, `Fwd:`, `Fw:` prefixes (case-insensitive)
- Lowercase, split on whitespace
- Remove stop words: `{"the", "a", "an", "to", "for", "with", "on", "in", "at", "by", "up", "and", "or", "of", "is", "it", "re", "fwd", "fw"}`
- Return set of remaining words

`_find_duplicate(session, tenant_id, user_id, account_id, title, cutoff_hours=48) -> tuple[UUID | None, str | None]`:
- Normalize the candidate title. If fewer than 2 meaningful words, return (None, None) (skip dedup)
- Query open tasks: `status NOT IN ('done', 'dismissed')`, same `tenant_id` and `user_id`, `created_at > now() - cutoff_hours`
- If `account_id` is not None, filter by same `account_id` first. If no match, fall through to all-user query (cross-source dedup).
- For each candidate task: normalize its title, compute overlap = `len(new_words & existing_words)`, shorter = `min(len(new_words), len(existing_words))`. If `shorter > 0 and overlap / shorter >= 0.5`, return `(task.id, task.title)`.
- Return (None, None) if no match
- **Note:** Returns a tuple of (id, title) so the caller can populate `duplicate_of_title` in tasks_detail.

  - `metadata_ = candidate["metadata"]`

```python
# --- Channel Task Extraction (ETL-01) ---
try:
    await _append_event_atomic(factory, run_id, {
        "event": "stage",
        "data": {"stage": "extracting_channel_tasks", "message": "Extracting tasks from email..."},
    })
    channel_summary = await extract_channel_tasks(factory, tenant_id, user_id)
    stage_results["channel_tasks"] = channel_summary
    created = channel_summary.get("tasks_created", 0)
    dupes = channel_summary.get("duplicates_found", 0)
    msg = f"Extracted {created} task(s) from email"
    if dupes:
        msg += f" ({dupes} possible duplicate(s))"
    await _append_event_atomic(factory, run_id, {
        "event": "stage",
        "data": {"stage": "extracting_channel_tasks", "message": msg},
    })
except Exception as e:
    logger.error("Channel task extraction failed: %s", e)
    await _append_event_atomic(factory, run_id, {
        "event": "stage",
        "data": {"stage": "extracting_channel_tasks", "message": f"Channel task extraction skipped: {e}"},
    })
```
