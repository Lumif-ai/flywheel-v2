Create `VoiceProfileSettings.tsx` following the exact patterns from `TenantSettings.tsx` (inline edit + save) and `GranolaSettings.tsx` (mutation + toast + integration status).

**TypeScript interface:**
```typescript
interface VoiceProfile {
  tone: string | null
  avg_length: number | null
  sign_off: string | null
  phrases: string[]
  formality_level: string | null
  greeting_style: string | null
  question_style: string | null
  paragraph_pattern: string | null
  emoji_usage: string | null
  avg_sentences: number | null
  samples_analyzed: number
  updated_at: string
}
```

**Data fetching:**
- `useQuery` with key `['voice-profile']` calling `api.get<VoiceProfile | null>('/email/voice-profile')`
- `useMutation` for save: `api.patch('/email/voice-profile', updates)` with `onSuccess` invalidating `['voice-profile']` and `toast.success('Voice profile updated')`
- `useMutation` for reset: `api.post('/email/voice-profile/reset')` with `onSuccess` invalidating `['voice-profile']` and `toast.success('Re-learning your voice from sent emails...')`

**Component states:**

1. **Loading state:** Show `Loader2` spinner with `animate-spin` (same pattern as other settings components).

2. **No profile state** (profile is null): Show card with message "No voice profile yet. Connect Gmail to get started." -- no edit/reset controls visible.

3. **Profile exists state:**
   - Header: "Learned from {samples_analyzed} emails" as muted text below the section title
   - Display all 10 fields in a clean layout (use a definition-list style or labeled rows):
     - Field label (from the display mapping in research: "Tone", "Sign-off", "Average Length", etc.)
     - Field value as descriptive text
     - For `phrases`: render as comma-separated string or small badges/chips
     - For null values: show "Not detected" in muted text
   - **Editable fields (tone, sign_off):** Render as text inputs (not plain text) with local `useState` state initialized from the profile data. Use `useEffect` to sync local state when profile data changes.
   - **Save button:** Below the editable fields. Disabled when `saveMutation.isPending` or when neither field has changed from the fetched value. Shows `Loader2` spinner when saving.
   - **Reset & Relearn button:** At the bottom, styled as destructive/outline variant. Opens a `Dialog` confirmation with title "Reset & Relearn Voice Profile?" and body explaining this will delete the current profile and re-analyze sent emails. Confirm button triggers `resetMutation.mutate()`. Dialog closes on success via `setResetDialogOpen(false)`.

4. **Re-learning state:** After reset succeeds, the query invalidation causes a refetch. The GET will return null (profile was deleted), so the component shows a modified empty state: "Re-learning your voice from sent emails... This usually takes about a minute." This is handled naturally -- when profile is null AND resetMutation recently succeeded (track with a `useRef` or simple state flag), show the re-learning message instead of the "Connect Gmail" message.

**Imports:** Use `useQuery`, `useMutation`, `useQueryClient` from `@tanstack/react-query`. `toast` from `sonner`. `Loader2` from `lucide-react`. `Dialog`, `DialogContent`, `DialogHeader`, `DialogTitle`, `DialogDescription`, `DialogFooter` from `@/components/ui/dialog`. `Button` from `@/components/ui/button`. `Input` from `@/components/ui/input`. `api` from `@/lib/api`.

**Styling:** Use Tailwind classes consistent with other settings components. Cards use `rounded-lg border p-6` pattern. Labels use `text-sm font-medium`. Values use `text-sm text-muted-foreground`. Section spacing with `space-y-4` or `space-y-6`.
